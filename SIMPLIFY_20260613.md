# Codebase Simplification Report — Removal of SSL/TLS and Kerberos 5 Support

**Date:** 2026-06-13
**Tree:** C-Kermit 10.0 Beta.12 (`/Users/kenji/src/cpp-kermit`)
**Scope:** Remove all SSL/TLS and Kerberos 5 code. As a verified consequence (see
[Scope decision](#scope-decision)), the entire Telnet authentication/encryption stack
went with it: Kerberos 4, Stanford SRP, GSSAPI, and the DES/CAST Telnet encryption
layer. Linux platform code untouched, as instructed (only the `linux+ssl`/`+krb*`/`+srp`
*build-target variants* were removed; all plain `linux*` targets remain intact).

## Method

1. **Analysis** was performed by a `c-expert` agent, which produced the macro
   classification, the file-deletion list, the makefile target inventory, and the
   risk list (e.g., the unconditional `TELOPT_AUTHENTICATION` constant uses in
   `ckctel.c` that must NOT be touched).
2. **Whole-file deletions** of the 8 security modules/headers.
3. **Conditional-block removal** with `/usr/bin/unifdef -m` (in-place) over all
   remaining `*.c *.h *.w`, in two passes, with per-file error checking (no parse
   errors occurred).
4. **Makefile surgery**: scripted deletion of every `+ssl`/`+krb*`/`+srp` target
   block on every platform, plus surgical edits to the live link/clean/dependency
   rules and removal of the crypto library variables.
5. **Regeneration of `ckcpro.c`** from the cleaned `ckcpro.w` (`make wart && make
   ckcpro.c`), then rebuild and smoke test on macOS.

## Scope decision

The request was "SSL and Kerberos 5"; the tree also contained Kerberos 4, Stanford
SRP, GSSAPI, and a Telnet DES/CAST encryption layer. The c-expert analysis showed
these are not independently viable once SSL and KRB5 are gone:

- On Unix, the only thing that enables any of this code is the makefile passing
  `-DCK_SSL` / `-DCK_KERBEROS -DKRB5` / `-DCK_AUTHENTICATION -DCK_SRP` etc. from a
  `+ssl`/`+krb*`/`+srp` target; `ckcdeb.h` then derives `CK_AUTHENTICATION` →
  `CK_SECURITY` and friends. No plain platform target defines any of them.
- KRB4 was only ever enabled together with KRB5 (`linux+krb5+krb4...`) or on dead
  platforms (SunOS 4.1, Solaris 2.x); SRP targets link `-lsrp -lgmp -ldes -lkrypto`
  (Stanford SRP libraries, uninstallable on any current system); the Telnet
  encryption layer (`ck_crp.c`) requires a DES library from Kerberos or OpenSSL.

So the whole stack was removed as dead code. **Kept, explicitly out of scope:**
`CK_PAM` (PAM), shadow passwords (`linux+shadow+pam` target intact), `CK_SOCKS`
(SOCKS proxy), `ZLIB` source blocks, and the Telnet option *constants*
`TELOPT_AUTHENTICATION`/`TELOPT_ENCRYPTION`/`TELOPT_START_TLS` in `ckctel.h`
(plain protocol numbers 37/38/46, referenced by unconditional negotiation code).

## Preprocessor symbols removed (unifdef)

**Pass 1 — treated as never defined (`-U`), 31 symbols:**

- Core stack: `CK_SSL`, `CK_AUTHENTICATION`, `CK_SECURITY`, `CK_ENCRYPTION`,
  `CK_DES`, `CK_CAST`, `LIBDES`, `LIBDES_USE_OPENSSL`
- Kerberos: `CK_KERBEROS`, `KRB5`, `KRB4`, `KRB524`, `KRB5_U2U`, `GSSAPI`,
  `HEIMDAL`, `MIT_CURRENT`
- SRP: `CK_SRP`
- Derived/auxiliary: `CK_FORWARD_X` (X11 forwarding over Telnet auth), `SSLDLL`,
  `CRYPT_DLL`, `TLS_VERIFY`, `SSL_KRB5`, `ENCRYPTION`, `DES_ENCRYPTION`,
  `CAST_ENCRYPTION`, `CAST_OR_EXPORT`, `CAST_EXPORT_ENCRYPTION`
- OpenSSL version selectors: `OPENSSL_097`, `OPENSSL_098`, `OPENSSL_100`,
  `OPENSSL_300`

Unlike the Windows/VMS removal, there was **no `-D` trap**: none of these macros is
true on a plain Unix build, and none appears in compound `#if defined(A) || defined(B)`
expressions, so unifdef performed pure block deletion.

**Pass 2 — residuals found by post-pass scan (`ckcftp.c` only):**

- `FTP_GSSAPI`, `FTP_KRB4`, `FTP_KRB5`, `FTP_SRP`, `FTP_SECURITY` — their `#define`
  sites were inside pass-1-stripped blocks, making them provably never defined; a
  targeted `unifdef -m -U... ckcftp.c` removed FTP-over-TLS, GSSAPI/Kerberos FTP
  authentication, and FTP SRP.

**Deliberately untouched:** `CK_PAM`, `CK_SOCKS`, `ZLIB`, shadow-password code, the
`TELOPT_*` constants in `ckctel.h`, the harmless `#define ck_ssleay_is_installed() (0)`
fallback stub in `ckuus6.c`, and `-DNOKERBEROS` no longer exists (its one makefile
use was deleted; no source file still tested it).

## Makefile (9,514 → 7,318 lines)

Unlike the Windows/VMS removal, this one required substantial makefile work,
because the security stack was wired in via build targets:

- **110 crypto target blocks deleted** across all platforms: every
  `+ssl`/`+openssl`/`+ibmssl`/`+krb4`/`+krb5`/`+srp` variant (live platforms
  `macos`/`linux`/`freebsd`/`netbsd`/`openbsd` and obsolete ones — AIX, HP-UX,
  IRIX, Solaris, SunOS, SCO, Tru64, UnixWare), plus the `krbmit`/`srpmit` link
  helpers, the `oldmacosx103+secure` target, and the five `redhat71`–`redhat9`
  targets (which chained to the deleted `linux+krb5+krb4+openssl+zlib+shadow+pam`).
- **Live link/clean/dependency rules edited**: `ckuath.$(EXT)`, `ck_crp.$(EXT)`,
  `ck_ssl.$(EXT)` removed from the `xermit` prerequisite and link lists (the target
  all live platform builds use) and from `clean`; their dependency rules deleted.
  (`wermit` and `mermit` never listed them.)
- **Crypto variables removed**: `srproot`, `sslroot`, `K4LIB/K4INC`, `K5LIB/K5INC`,
  `SRPLIB/SRPINC`, `SSLLIB/SSLINC`, `CERTDIR`, and their `show:` echoes.
- **Install target**: the `ca_certs.pem` certificate-installation section removed
  (no `ca_certs.pem` exists in the tree).
- **Stale comments pruned**: the "SECURE TARGETS" documentation section, the secure
  target catalogs, OpenSSL version notes, and per-target crypto mentions.
- A dispatch case arm in `hpux1000t` mapping `*+ssl` to a deleted target was removed.

`make -n macos` and a real build verify the makefile still parses and links.
