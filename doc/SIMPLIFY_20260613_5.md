## 2. Method — precise scoping

1. **Classification by description.** The SCO family splits into SCO **Xenix**
   (`sco86`/`sco286*`/`sco386*`/`sco3r2*`/`sco234*`) and SCO **UNIX/OpenServer**
   (`sco3r22*`, `sco32v*`, `sco_osr600`). Only the Xenix branch was removed; the SVR3.2
   SCO UNIX targets are *not* Xenix and were kept. `altos3` ("Altos UNIX System III")
   was likewise kept — it is UNIX System III, not Xenix.

2. **Dispatch closure to a fixpoint** with a **dangling-reference check**, identical to
   the method in `SIMPLIFY_20260613_4.md`: starting from the seed of Xenix/ancient-BSD/
   BSDI/SunOS targets, repeatedly add any target that invokes a target already marked
   for deletion, and confirm that **no kept target references a removed one**. The final
   check reported zero dangling references.

3. **Stanza-accurate deletion** (comment block + header + recipe + one trailing blank)
   by an exact-boundary script; then **entry-aware menu pruning** that removes whole
   dead menu entries (including continuation lines) and skips genuine mixed entries.

4. **Menu rewrites.** Three mixed menu entries were rewritten by hand to keep their
   still-valid alternatives rather than deleted:
   - 4.3-Reno: `"make bsd43" or "make bsd44" or "make bsd44c"` → `"make bsd44" or "make bsd44c"`.
   - IBM RT PC ACIS 4.3: `"make rtacis" or "make bsd KFLAGS=-DNOANSI"` → `"make bsd KFLAGS=-DNOANSI"`.
   - Mach 2.6: `"make bsd42" or "make bsd43"` → `"make bsd42"`.
   (The `sco3r2`/`sco3r2x` entry was deleted outright: `sco3r2x` is not a real target.)

## 3. Deliberately KEPT

- **`bsd` and `bsd42`** — the generic 4.2/4.3BSD base recipes. They are **load-bearing
  for out-of-scope platforms**: the SGI IRIX (`iris`, `irix40`, `irix40ypc`), MIPS
  (`mipstcpc`), and DNIX (`dnix5r3net`, `dnix5r3ansinet`, `dnixnetc`) targets all
  dispatch through `bsd` → `bsd42`. Removing them would have cascaded into SGI/MIPS/DNIX,
  exceeding the "Xenix and ancient BSD" scope.
- **`bsd44` / `bsd44c`** — the 4.4BSD-Lite lineage base (macOS-adjacent), kept by
  request; several retained menu entries point users here.
- **SCO UNIX / OpenServer** (`sco3r22*`, `sco32v4*`, `sco32v5*`, `sco32v50x*`,
  `sco-odt30`, `sco_osr600`/`sco600`) and `altos3` — System V, not Xenix.
- All modern targets: `linux`, `macos`/`macosx`, `freebsd*`, `netbsd*`, `openbsd`,
  `mirbsd`, plus build infrastructure (`xermit`, `wermit`, `wart`) — confirmed present.
