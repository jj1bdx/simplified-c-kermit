## Background

`_PROTOTYP` was a portability alias from C-Kermit's pre-ANSI era. After the
non-ANSI / K&R removal (`SIMPLIFY_20260613_2.md`) its definition collapsed to a
single unconditional one-liner in `ckcdeb.h`:

```c
#define _PROTOTYP(func, parms) func parms
```

so `_PROTOTYP(int ttclos, (int));` is identical to `int ttclos(int);`. The
earlier pass deliberately left the ~1,175 call sites in place (high-churn,
zero-benefit at the time); this pass performs that expansion now that a
reliable, verifiable method is in hand.

## Method

1. **Baseline build** captured for verification:
   `make clean && rm -f wermit && SOURCE_DATE_EPOCH=1700000000 make linux`,
   binary copied aside.

2. **Mechanical expansion** via a one-shot, comment/string-aware,
   paren-balanced Python script (run once, then removed — not committed). For
   each real `_PROTOTYP(` invocation it:
   - tracks `/*…*/`, `//`, and string/char-literal state across the whole file,
     so prose and commented-out occurrences are skipped;
   - tracks paren depth from the opening `(`; the **first depth-1 comma** splits
     `arg1` (func) from `arg2` (parms), and depth returning to 0 marks the
     macro's close;
   - **guards:** transform only if a top-level comma exists *and* `arg2` begins
     with `(` — this skips the `#define` line (`arg2` = `parms`) and the prose
     `_PROTOTYP()` (no comma);
   - replaces *only* the macro span with `arg1.strip() + arg2.strip()`,
     preserving everything after it verbatim (trailing `;`, ` = NULL;`,
     trailing comments);
   - for multi-line invocations, re-indents continuation lines under the new
     parameter-list `(`.

3. **Definition removal** from `ckcdeb.h`: deleted the `#define _PROTOTYP`, its
   explanatory comment block, and the already-dead empty `#ifdef __STDC__` /
   `#endif` pair (the last of the residue noted in `SIMPLIFY_20260613_2.md`).

4. **Header reformatting:** ran `clang-format -i *.h`. The headers were already
   clang-format-clean (commit `459e860`), so this re-wrapped *only* the expanded
   prototype lines (cleanly handling the multi-line ones). `*.c` and `ckcpro.w`
   were left mechanical — the `.c` files have never been clang-formatted
   (25–42-line diffs each), so formatting them would add large unrelated churn.

5. **Protocol module regenerated:** `make wart && make ckcpro.c` (the 9
   `ckcpro.w` invocations are all single-line; `wart` copies them through
   verbatim, so the expansion propagates into `ckcpro.c`).

## Edge cases handled

- **Function-pointer variable declarations** reusing the macro, with nested
  parens in `arg1` and a trailing initializer:
  `_PROTOTYP(CHAR(*sxo), (CHAR)) = NULL;` → `CHAR(*sxo)(CHAR) = NULL;`
  (`ckcfns.c:1775`, `ckufio.c:2496`, `ckusig.c`, `ckuxla.c`, `ckuus4.c`,
  `ckudia.c`, `ckcker.h`). The depth-1-comma split keeps `arg1`'s `(*name)`
  intact; the trailing text is preserved.
- **Function-pointer parameters** in `arg2`
  (`_PROTOTYP(VOID conint, (SIGTYP (*)(int), SIGTYP (*)(int)));`,
  `int (*)(char)`) — balanced as text.
- **Multi-line invocations** (both the wrapped-params and `func,\n (parms)`
  styles), **inline `/* timo */` parameter comments**, **named params**,
  **`static`/multi-word return types**, **array `[]` declarators** (in `arg2`).
- **Skipped (intentionally):** the `#define` and prose in `ckcdeb.h`, the
  commented-out invocations at `ckcker.h:1057` and `ckcker.h:1268`, the prose
  reference in `ckcfnp.h:18`, and `_PROTOTYP` text inside string literals.

## Why this is provably behaviour-preserving

The replacement text is exactly what the preprocessor already substituted for
each invocation. Inter-token whitespace is insignificant (the macro's literal
`func parms` space vs. our no-space join makes no difference), and because
`arg2` always begins with `(`, no accidental token pasting can occur (`name(`
and `)(` were always two tokens). Therefore the token stream handed to the
compiler is unchanged on every `#ifdef` path — not only the Linux-active ones —
and the build is byte-identical.

## Notes / follow-ups

- The cross-platform argument is textual and `#ifdef`-independent, so the
  byte-identical Linux build covers the macOS/BSD-only prototype lines too; an
  optional `gcc -E -DMACOSX10` / `-DBSD44` token-diff could confirm a
  macOS/BSD build directly.
- The portability aliases `VOID`, `CKVOID`, and `CONST` remain defined and in
  use; only `_PROTOTYP` was expanded and retired here.
- The throwaway expansion script was not committed; nothing other than the
  source edits is left in the tree.
