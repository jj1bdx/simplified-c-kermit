# Codebase Simplification Report 2 — Removal of Android Support

**Date:** 2026-06-12
**Tree:** C-Kermit 10.0 Beta.12 (`/Users/kenji/src/cpp-kermit`)
**Scope:** Remove all Android support. Linux, macOS, and generic Unix code untouched.
Follows the Windows/OS/2/VMS removal documented in `SIMPLIFY_20260612.md`.

## Summary

Android support in this tree was small and cleanly separated: a standalone NDK build
file, a readme, `#ifdef ANDROID` conditionals at four source sites, and a stub
makefile target. All of it is now removed.

| Metric | Before | After | Delta |
|---|---|---|---|
| Source lines (`*.c *.h *.w`, excl. generated `ckcpro.c`) | 276,983 | 276,968 | −15 |
| Android-only files deleted | 2 (59 lines) | 0 | −59 |
| Source files edited (`unifdef -m`) | — | 4 | |
| Makefile lines removed (comments + stub target) | 6 | 0 | −6 |

Build verification on macOS: **success**. The `wermit` binary is byte-for-byte the
same size as before the removal (2,456,584 bytes) with an identical warning count
(27) — expected, since `ANDROID` is derived only from the compiler-predefined
`__ANDROID__`, which never exists on macOS, so all removed code was dead there.

A full backup of the tree as it stood before this change was made first:
`/tmp/cpp-kermit-backup-20260612-2.tar.gz`

## Method

1. **Analysis** by a `c-expert` agent: located every `ANDROID`/`__ANDROID__`
   occurrence, verified branch survival for Linux/macOS at each site, confirmed no
   `#if` expression mixes ANDROID with Linux/macOS macros, and confirmed `ckcdeb.h`
   derives nothing else (no feature macros) from ANDROID.
2. **Deletion** of the Android-only files.
3. **`unifdef -m -U__ANDROID__ -UANDROID`** on the four affected sources (in-place,
   no parse errors).
4. **Hand-edit** of the makefile (unifdef does not process makefiles).
5. **Rebuild and smoke test** on macOS.

## Files deleted (2)

- `android.mk` (31 lines) — Android NDK build script (`LOCAL_PATH`/`CLEAR_VARS`/
  `LOCAL_SRC_FILES`); consumed only by the NDK, never by the Unix `makefile`
- `android.txt` (28 lines) — Android build instructions referencing `android.mk`

## Source sites removed (4 files, −15 lines)

| File | Site | What it did | Surviving branch |
|---|---|---|---|
| `ckcdeb.h` (−5) | `#ifdef __linux__` → `#ifdef __ANDROID__ → #define ANDROID` | The only derivation of `ANDROID` | Plain `LINUX` define block — unchanged for Linux/macOS |
| `ckufio.c` (−4) | two `#ifndef ANDROID` guards around `pw_gecos` access (bionic libc lacks `pw_gecos`) | Excluded gecos code on Android | gecos code **kept** (always compiled on Linux/macOS) |
| `ckutio.c` (−2) | inner `#ifndef ANDROID` around `ctermid(cttnam)` under `#ifdef SVORPOSIX` | Excluded `ctermid()` on Android | `ctermid()` call **kept**; load-bearing `SVORPOSIX` guard untouched |
| `ckuver.h` (−4) | `#ifdef ANDROID` `HERALD " Android"` `#else` `HERALD " Linux"` | Android herald string | `#else` branch correctly kept: `HERALD " Linux"` (verified post-edit) |

The `ckuver.h` site was the one inverted case (Linux code in the `#else` branch);
unifdef resolved it correctly.

## Makefile edits (3, all hand-applied)

- Removed comment line "For Android, use android.mk" (separate-makefiles list, was line 89)
- Removed comment line `"make -f android.mk" (separate makefile) for Android` (common-targets list, was line 150)
- Removed the stub `android:` target (was lines 6436–6438), which only echoed a
  pointer to the deleted `android.mk`; no other target depended on it

No `-DANDROID` ever appeared in any `CFLAGS`/`KFLAGS`/`LIBS`, so no build-relevant
makefile logic was involved.
