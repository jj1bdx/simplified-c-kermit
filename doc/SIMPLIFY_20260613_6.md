## 3. Method

Mirrors `SIMPLIFY_20260613_4/5`, with one important refinement:

1. **Enumerate** every column-0 rule header (excluding `=` variable assignments and
   `$(...)`-named dependency rules); `REMOVE = all − KEEP − infrastructure`.
2. **Dispatch-closure + dangling-ref check** (now also matching lowercase `make` dispatch):
   confirmed **zero** real `KEEP → REMOVE` references. (`makewhat` only *echoes* `aix`/
   `solaris10` as help text — fixed, not a dependency. `linux`/`gnu-linux` genuinely
   dispatch to `linuxa`, so `linuxa` was promoted to KEEP.)
3. **Stanza-accurate deletion**: preceding comment block + header + recipe + one trailing
   blank.
4. **Menu rewrite**: the ~540-line top-of-file "Complete list (alphabetical)" platform menu
   was replaced with a concise Linux/macOS/BSD menu; stale `make aix`/`make macosx`/
   lint-target examples elsewhere in the header comments were updated.

### Methodological note — recipe blank-line handling

GNU make ignores **blank lines and comment lines interspersed between tab-prefixed recipe
lines**; such a recipe (e.g. the HP-UX targets, which chain several `$(MAKE)` invocations
separated by blank lines) is a single recipe. The first deletion attempt used a parser that
stopped a recipe at the first blank line, which deleted the header but **orphaned the rest
of the recipe**, gluing an HP-UX recipe fragment onto a kept link target and breaking
`make macos`. The parser was corrected to treat interspersed blank/comment lines as part of
the recipe (the recipe ends at the last tab-line before the next header), the tree was
restored from backup, and the strip was re-run. Future makefile-stanza work must use this
blank-line-aware boundary detection.
