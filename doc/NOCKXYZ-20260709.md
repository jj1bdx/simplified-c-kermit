# Removal of XMODEM/YMODEM/ZMODEM support (NOCKXYZ always defined)

Date: 2026-07-10 (branch `remove-xyz`)
Requested as: "Remove XMODEM/YMODEM/ZMODEM code altogether, namely by assuming the
compilation flag NOCKXYZ is ALWAYS defined."

## Summary

| Metric | Before | After | Delta |
|---|---|---|---|
| Source lines (`*.c *.h *.w` + makefile) | 221,059 | 220,275 | −784 |
| Diff over 16 source files | | +44 / −826 | (plus README.md +2) |
| `wermit` binary (`make linux`, x86_64) | 2,664,904 | 2,655,672 | −9,232 bytes |

`NOCKXYZ` is now hard-wired in `ckcdeb.h`, so `CK_XYZ` — the macro that actually
gated all XMODEM/YMODEM/ZMODEM code — is never defined. The never-enabled
`XYZ_INTERNAL` / `XYZ_DLL` variants were swept out with it.

**This removed a live feature, not dead code.** No makefile target ever defined
`NOCKXYZ`, so every default build (linux/macos/BSD) had `CK_XYZ` auto-defined via
`ckcdeb.h`'s `#ifndef NOCKXYZ → #ifdef UNIX → #define CK_XYZ` arbitration:
`SET PROTOCOL XMODEM/YMODEM/ZMODEM` (dispatch to external `rx`/`sx`/`rb`/`sb`/
`rz`/`sz` programs via the REDIRECT machinery) and ZMODEM autodownload detection
in CONNECT mode worked until this change. The Kermit protocol itself is untouched.

## Macro logic and equivalence proof

Pre-change semantics, established empirically before any edit (`gcc -E -dM` full-macro
dumps over three platform flag sets — linux, MACOSX10, BSD44 — for `ckcdeb.h` alone and
with `ckcnet.h`, six configurations): adding `-DNOCKXYZ` changes **exactly two**
macros — `NOCKXYZ` appears, `CK_XYZ` disappears. `XYZ_INTERNAL` (built-in protocol
engines) and `XYZ_DLL` (loadable engine) are defined by nothing in this tree, on any
platform; every `#ifdef XYZ_INTERNAL` body was already dead code.

Unlike `NODEPRECATED` (see `doc/NODEPRECATED-20260709.md`), this flag does exactly
what its name promises — no backstops, no scope surprises.

Post-change proof, same six configurations:

- new tree with **no** flag vs. old tree with `-DNOCKXYZ`: the only difference is the
  definition form of `NOCKXYZ` itself (`#define NOCKXYZ 1` from the command line vs.
  the header's bare `#define NOCKXYZ`);
- new tree **with** `-DNOCKXYZ` vs. old tree with `-DNOCKXYZ`: byte-identical dumps.

## Method

1. **ckcdeb.h hand-edited** (excluded from the mechanical pass — `unifdef -DNOCKXYZ`
   would delete the very `#ifndef NOCKXYZ #define NOCKXYZ #endif` trio being
   installed): the CK_XYZ/XYZ_INTERNAL/XYZ_DLL arbitration block (old lines
   2915–2942) replaced by a dated comment plus the hard-wire trio; the two now-redundant
   `NOCKXYZ` define-trios inside the `NOXFER` (old 220–222) and `NOICP` (old 262–264)
   cascades deleted.
2. **Mechanical pass**: `unifdef -DNOCKXYZ -UCK_XYZ -UXYZ_INTERNAL -UXYZ_DLL -m` over
   the 16 other files with references. 15 modified; `ckcnet.c` untouched (its only
   hit is `-d=CK_XYZ` inside a stale OSK cross-build recipe comment, left as-is).
3. **Hand fix** (the one hazard the c-expert audit confirmed, by real unifdef + compile
   with the project's exact flags): `static int zmdlok` in `ckucns.c` and `ckucon.c`
   was declared by *both* branches of its `#ifdef CK_XYZ`, so unifdef correctly left
   the declaration while deleting every use — deleted the orphaned declaration in both
   files. (Silent in `make linux`, which passes no `-W` flags; would have been a
   `-Wunused-variable` under `-Wall` and a clangd diagnostic.)
4. `ckcpro.w` regenerated: `make wart && make ckcpro.c`, then `make linux`.
5. All touched `.c`/`.h` files re-verified clang-format-stable (11 needed a reformat
   pass — expected comment-column drift from guard un-nesting).

## What was removed

- **ckcker.h**: `PROTO_X`/`PROTO_XC`/`PROTO_Y`/`PROTO_G`/`PROTO_Z`/`PROTO_O`
  constants; `NPROTOS` collapses 7 → 1 (`PROTO_K` alone remains); the guarded
  `zstart()` prototype.
- **ckcmai.c**: the six non-Kermit entries of the protocol-parameter table
  `ptab[NPROTOS]`; the external-command `initproto()` calls for X/Y/Zmodem (the
  Kermit-only `#else` branch is what compiles now).
- **ckuus3.c**: the `protos[]` keyword table shrinks to `"kermit"`; the `"protocol"`
  keyword in `tstab[]` (so `SET TRANSFER PROTOCOL` is gone); the ZMODEM branch of
  `SET WINDOW-SLOTS`; the `"xyzmodem"` feature row flips to "absent" for
  `IF FEATURE`.
- **ckuusr.c**: the `SET EXTERNAL-PROTOCOL` keyword; MSEND's `/protocol` switch;
  eight protocol-restriction runtime checks (BYE/FINISH/REMOTE/SERVER/PSEND/… "works
  only with Kermit" guards, now vacuous).
- **ckuus2.c / ckuus5.c**: long XMODEM/YMODEM/ZMODEM `HELP SET PROTOCOL` text (short
  Kermit-only `#else` text now live); `SEND /PROTOCOL` help paragraph; ZMODEM clause
  in `SET TERMINAL AUTODOWNLOAD` help; SHOW FEATURES flips from " External XYZMODEM
  protocol support" to the `#ifndef CK_XYZ` line " No built-in XYZMODEM protocols"
  (the single `#ifndef CK_XYZ` block in the tree that genuinely goes live).
- **ckuus4.c**: the external-protocol section of `SHOW PROTOCOL` (`shoparp()`),
  including the `shoextern()` call; `VN_PROTO` (`\v(protocol)`) now returns the
  Kermit `#else` value.
- **ckuus6.c / ckuus7.c**: GET/XMODEM disambiguation; `protocol != PROTO_K` checks;
  XMODEM 128/1024 packet-length validation. SEND's `/protocol:` switch remains but
  becomes invisible (`CM_INV`) per the original `#else` branch — it still parses and
  only accepts `kermit`.
- **ckucns.c / ckucon.c / ckcfn2.c**: ZMODEM autodownload — the `zstart()`
  start-of-packet scanner, its call sites in both CONNECT implementations and in
  `autodown()`, the `k == PROTO_Z` handling and 3×CAN packet-damaging branch, and the
  APC "set proto xmodem…" string builder. Kermit autodownload (`kstart()`,
  `CK_AUTODL`) is untouched.
- **ckcpro.w**: the entire non-Kermit protocol-dispatch branch (build external
  command line from `ptab[].p_?_?cmd` and run it via `ttruncmd()`), and the XMODEM
  filename-substitution special case.
- **ckuusx.c**: an `#ifdef XYZ_INTERNAL` curses status-bar "Protocol:" label that had
  never been compiled in any build of this tree (dead under the never-defined
  `XYZ_INTERNAL`, independent of this change).

## Deliberately NOT removed

`setextern()`, `shoextern()`, `extprotab[]`, and the `exp_handler`/`exp_timo`/
`exp_stderr` globals in `ckuus3.c` (plus `case XYEXTRN:`) are gated by
`CK_REDIR`/`NOPUSH`/`XYZ_INTERNAL` — **not** `CK_XYZ` — and `exp_handler`/`exp_timo`
are read by `ttruncmd()` in `ckutio.c`, which also implements the generic REDIRECT
command. They stay compiled. Consequences: `exp_*` are permanently stuck at their
defaults (automatic handler, 12 s timeout) now that `SET EXTERNAL-PROTOCOL` is gone,
and `shoextern()` is compiled but uncalled (external linkage, so no warning). Do not
"clean these up" as XYZ leftovers — that would break REDIRECT.

## Behavior changes (user-visible)

| Command | Before | After |
|---|---|---|
| `SET PROTOCOL ?` | kermit, x/xmodem, xmodem-crc, y/ymodem, ymodem-g/g, zmodem, other | kermit only (`?No keywords match - zmodem`) |
| `SET TRANSFER PROTOCOL` | synonym for SET PROTOCOL | keyword gone (`?No keywords match - protocol`) |
| `SET EXTERNAL-PROTOCOL` | sets handler/timeout for spawned protocol programs | keyword gone |
| `SHOW PROTOCOL` | non-Kermit: "Executed by external commands:" + rz/sz strings + handler/timeout lines | Kermit block only |
| `SHOW FEATURES` | " External XYZMODEM protocol support" | " No built-in XYZMODEM protocols" |
| `HELP SET PROTOCOL` | long form with XMODEM/YMODEM/ZMODEM + s1…s6 | short `SET PROTOCOL KERMIT [ s1 [ s2 [ s3 ] ] ]` |
| CONNECT autodownload | recognizes Kermit packets **and** ZMODEM `rz` start sequences | Kermit packets only; incoming `sz` output just displays |
| `SEND /PROTOCOL:` | visible switch, all protocols | hidden (`CM_INV`), parses, kermit only |
| `MSEND /PROTOCOL:` | visible switch | removed outright (original code had no CM_INV fallback here) |
| REDIRECT | works | works, unchanged |

## Verification

- `make wart && make ckcpro.c && make linux`: exit 0, zero warnings (all
  "error/warning" grep hits in the log are `-DUSE_STRERROR` command echoes).
- Macro equivalence proof: 6/6 configurations pass (see above).
- clang-format: all 15 touched `.c`/`.h` files stable (`clang-format FILE | diff FILE -`
  empty).
- Runtime smoke tests (each probe in its own `wermit -Y` invocation — batch mode
  aborts on first command error): `set protocol zmodem` → `?No keywords match`;
  `set protocol kermit` + `show protocol` → Kermit block, no external-protocol lines;
  `set transfer protocol` / `set external-protocol` → keywords gone; `help set
  protocol` → short form; SHOW FEATURES → " No built-in XYZMODEM protocols";
  `send /protocol:kermit nosuchfile` → parses switch, fails on the file as expected;
  `help redirect` intact and `redirect echo hi` gives the normal "?SET LINE or SET
  HOST required first"; `PAUSE 2` = 2.01 s (signal machinery healthy).
- Hazard audit (c-expert, empirical): every `PROTO_X…PROTO_O` reference tree-wide was
  verified to sit inside `#ifdef CK_XYZ` — zero dangling references; unifdef output
  for all 15 real files compiled clean stand-alone with the exact `make linux` flags
  both before and after, the orphaned `zmdlok` being the only `-Wall` delta.

## Residuals (intentional)

- `ckcnet.c:1149` — `-d=CK_XYZ` inside a decades-old OSK cross-build recipe comment.
- `ckuus3.c:11391/11398` — `/* #ifdef CK_XYZ */ … /* #endif */` markers a previous
  author deliberately commented out around `case XYX_PRO:` (the SET TRANSFER PROTOCOL
  handler was intentionally unconditional; the case is now unreachable since the
  keyword is gone, but compiles silently).
- `ckuus3.c:6869` — prose comment describing the old CK_XYZ arrangement, kept as
  history above `protos[]`.
- Dead runtime checks that a `-DNOCKXYZ` build always compiled and which strict
  semantics therefore keeps: `ckuus7.c` "Use SET SEND PACKET-LENGTH for XYZMODEM"
  printf (unreachable — `protocol` can no longer be `!= PROTO_K`), `ckuusr.c`
  ADD SEND-LIST external-protocol bail-out, `ckuus4.c` `dostat()` external-protocol
  bail-out, and the `setproto()` external-command prompting path in `ckuus3.c`
  (reachable only through `protos[]`, which now only maps `kermit`).
- `PROTO_K`, `struct ck_p`, `ptab[]`/`protv` and `NPROTOS` (= 1) remain — they are the
  live Kermit protocol bookkeeping, not XYZ leftovers.

## Deviation from strict flag semantics

One, matching the spirit of the RLOGIN pass: the orphaned `static int zmdlok = 0;`
declarations in `ckucns.c`/`ckucon.c` were deleted even though a `-DNOCKXYZ` compile
of the old tree kept them (unused). Pure dead code; no reference remains.
