# OS-Implementation Compilation Flags in C-Kermit (this tree)

Date: 2026-06-13
Scope: catalog of all OS-implementation-related preprocessor macros (operating
system / OS-family / OS-version / OS-capability selectors) used in the C-Kermit
10.0 Beta.12 source tree at `/Users/kenji/src/cpp-kermit`, after Windows, OS/2,
VMS, and Android support were removed. Pure feature flags (NONET, NOSSL,
NODEBUG, NOSPL, ...) are out of scope except where they exist specifically to
adapt to an OS.

## 1. How this was analyzed

- Tallied every `#ifdef` / `#ifndef` / `defined(...)` test across all `*.c` and
  `*.h` files (2482 distinct macro names tested; OS-related subset extracted).
- Tallied every `-D` flag in the authoritative `makefile` (~9514 lines) and
  cross-referenced against the code tests.
- Read the master header `ckcdeb.h` (6097 lines), which is included everywhere
  and performs nearly all OS-macro *derivation* (one macro implying another).
- Read the source of each remaining "live" platform target in the makefile —
  `macos`, `macos+ssl`, `linux`/`linuxa`, `freebsd`, `netbsd`, `openbsd` — to
  record exactly which OS flags they pass via `CFLAGS`/`KFLAGS`.

A key distinction: a macro being tested in the code does NOT mean it is reachable
from a buildable target. Likewise, the makefile retains *hundreds* of obsolete
per-platform targets (XENIX, SVR4, AIX, HP-UX, Plan9-ish, etc.). They are all
still "in the makefile" but only the Unix/Linux/macOS/BSD ones build on a
current system. Throughout this report, **"live target"** means the small set of
targets that actually build here today (`macos*`, `linux*`, `freebsd*`,
`netbsd*`, `openbsd*`).

## 2. Catalog of OS flags

### 2.1 Top-level OS-family selectors (umbrella macros)

These are the broad classifiers the code branches on most. Almost all are
*derived* in `ckcdeb.h` rather than set on the command line.

| Flag | Selects | Defined by | Representative use |
|------|---------|-----------|--------------------|
| `UNIX` | Any Unix-family OS | Derived in `ckcdeb.h` from ATTSV, ANYBSD, POSIX, V7, COHERENT, MINIX, SUNOS4, SOLARIS, etc. (only `-DUNIX` once in makefile) | everywhere (293 `#ifdef UNIX` sites): `ckutio.c`, `ckufio.c`, `ckcnet.c`, `ckcmai.c` |
| `POSIX` | POSIX.1 termios / signals / dirent | makefile (`linuxa`, Solaris, QNX targets) and derived (SOLARIS25, QNX) | `ckutio.c`, `ckcdeb.h`, `ckufio.c` |
| `ATTSV` | AT&T System V (any release) | Derived from SVR3/SVR4/HPUX/XENIX/OXOS/AUX/ATT7300/ATT6300/UTSV/ISIII | `ckutio.c`, `ckufio.c`, `ckcnet.c` |
| `ANYBSD` | Any Berkeley Unix | Derived from BSD4/BSD43/BSD41/BSD29 | `ckutio.c`, `ckufio.c` |
| `BSD4` | 4.x BSD (generic) | makefile + derived (NEXT, BSD41, BSD43, SUNOS4, AIX370 all imply it) | `ckutio.c`, `ckufio.c` |
| `BSD44` | 4.4 BSD termios/dirent base | makefile (freebsd/netbsd/openbsd) + derived (MACOSX, AIXESA imply it) | `ckutio.c`, `ckcnet.c`, `ckufio.c` |
| `SVR3` | System V Release 3 | makefile + derived (SVR4, HPUX9, MIPS, MINIX-ish, SV68/SV88 imply it) | `ckutio.c`, `ckufio.c` |
| `SVR4` | System V Release 4 | makefile + derived (SOLARIS, HPUX10, BSD44, DGUX540 imply it) | `ckutio.c`, `ckcnet.c` |

Note: `MAC` (Classic Mac OS / MPW) is entirely distinct from `MACOSX`. With the
Mac Classic toolchain gone, all 97 `#ifdef MAC` sites are dead.

### 2.2 Derived "common-feature" umbrella macros (capability rollups)

These exist specifically so OS-dependent code can select behavior shared by
several OS families. They are **always derived** in `ckcdeb.h`, never passed on
the command line directly (a few appear in makefile comments/obsolete targets).

| Flag | True when | Purpose |
|------|-----------|---------|
| `SVORPOSIX` | `ATTSV` OR `POSIX` | "System V or POSIX" — things SysV+POSIX share that BSD/V7 lack (e.g. `<termio.h>`/`<termios.h>` tty model). 528 termio-selector references in `ckutio.c`. |
| `SVR4ORPOSIX` | `SVR4` OR `POSIX` | Items common to SVR4 and POSIX only. |
| `BSD44ORPOSIX` | `BSD44` OR `POSIX` | Items common to 4.4BSD and POSIX (modern termios); also force-defined by IRIX64. Heavily used in `ckutio.c`. |
| `VMSORUNIX` | `UNIX` (VMS branch now dead) | Legacy; defined whenever UNIX is. |

`SVORPOSIX` / `BSD44ORPOSIX` are the real workhorses of `ckutio.c`'s terminal
driver: they pick the termios/termio/sgtty implementation.

| `NOSETBUF` | `setbuf()` unsafe / unneeded on this OS | derived (BSD44, SUNOS4, SOLARIS, OXOS) | `ckutio.c`, `ckcmai.c` |
| `HDBUUCP` | HoneyDanBer UUCP lockfile convention | makefile + derived (SOLARIS) | `ckutio.c`, `ckufio.c` |
| `CK_UTSNAME` | System has `uname()`/`struct utsname` | makefile/probed | `ckufio.c`, `ckuus*.c` |
| `FNFLOAT` | OS/libc has floating-point math funcs for function library | makefile (all live targets) | `ckuus*.c`, `ckclib.c` |
| `SYSTIMEH` / `SYSTIMEBH` | Include `<sys/time.h>` / `<sys/timeb.h>` | derived per-OS in `ckcdeb.h` | `ckcdeb.h`, `ckutio.c` |
| `POSIX_CRTSCTS` | POSIX-style RTS/CTS hardware flow ctl | derived (SOLARIS+POSIX) | `ckutio.c` |

## 3. Derivation chains in ckcdeb.h

`ckcdeb.h` is where most OS macros are *implied*. The major chains (read "X ⇒ Y"
as "if X is defined, Y is defined"):

**Apple / BSD core**
```
MACOSX10 ⇒ MACOSX ⇒ BSD44
BSD44    ⇒ SVR4, NOSETBUF, DIRENT          (4.4BSD = SysV+POSIX+4.3BSD mix)
__linux__ ⇒ LINUX
OPENBSD  ⇒ __OpenBSD__
__bsdi__ ⇒ BSDI
```

**BSD ladder ⇒ family**
```
NEXT33 ⇒ NEXT ⇒ BSD4
BSD41 ⇒ BSD4 ;  BSD43 ⇒ BSD4
BSD4  ⇒ ANYBSD ;  BSD29 ⇒ ANYBSD
ANYBSD ⇒ UNIX
```

**System V ladder**
```
SVR4 ⇒ ATTSV + SVR3
SVR3 ⇒ ATTSV
HPUX ⇒ ATTSV ;  XENIX ⇒ ATTSV ;  OXOS/UTSV/AUX/ATT7300/ATT6300/ISIII ⇒ ATTSV
ATTSV ⇒ UNIX
MIPS / SV68R3V6 / SV88R32 ⇒ SVR3
DGUX540 ⇒ SVR4 ;  DGUX540/DGUX430 ⇒ DGUX
```

**Solaris / SunOS**
```
SOLARIS11 ⇒ SOLARIS10 ⇒ SOLARIS9 ⇒ SOLARIS8 ⇒ SOLARIS7 ⇒ SOLARIS26
          ⇒ SOLARIS25 ⇒ SOLARIS (+POSIX)
SOLARIS24 ⇒ SOLARIS
SOLARIS   ⇒ SVR4, STERMIOX, SELECT, FNFLOAT, DIRENT, CK_NEWTERM,
            NOSETBUF, CK_CURSES, HADDRLIST, DYNAMIC, TCPSOCKET, HDBUUCP,
            UNIX, POSIX_CRTSCTS
SUNOS41 ⇒ SUNOS4 ⇒ UNIX, BSD4, BIGBUFOK, NOSETBUF, DIRENT, TCPSOCKET,
                   SAVEDUID, DYNAMIC, SVR3(*see note)
```
(*The SUNOS4 block in `ckcdeb.h` also force-defines `SVR3` in one branch — a
historical quirk for the built-in SunOS makefile entry.)

**HP-UX / IRIX / AIX / OSF version ladders**
```
HPUX9  ⇒ SVR3, HPUX, HPUX9PLUS
HPUX10 ⇒ SVR4, HPUX, HPUX9PLUS ;  HPUX1100 ⇒ HPUX1010 ; HPUX1020/1030 ⇒ HPUX1010
IRIX65 ⇒ IRIX64 ⇒ BSD44ORPOSIX + IRIX62 ⇒ IRIX60 ⇒ IRIX51 + IRIX52
        (any IRIXxx ⇒ IRIX)
AIX53 ⇒ AIX52 ⇒ AIX51 ⇒ AIX50 ⇒ AIX45 ⇒ AIX44 ⇒ AIX43 ⇒ AIX42 ⇒ AIX41
AIXPS2 ⇒ AIX370 ⇒ BSD4 ;  AIXESA ⇒ BSD44
OSF50 ⇒ OSF40 ⇒ OSF32 ⇒ OSF30 ⇒ OSF20 ⇒ OSF10
```

**SCO / Xenix / MINIX**
```
SCO_OSR505 ⇒ SCO_OSR504 ⇒ CK_SCOV5 ⇒ ANYSCO
UNIXWARE ⇒ ANYSCO
CK_SCO32V4 ⇒ ANYSCO, XENIX, SVR3, DIRENT, SVR3JC
XENIX ⇒ ATTSV
MINIX340/MINIX315/MINIX3/MINIX2 ⇒ MINIX ⇒ UNIX
```

**POSIX / V7 / Coherent ⇒ UNIX, plus capability rollups**
```
POSIX ⇒ UNIX, DIRENT (unless SDIRENT), NOFILEH, _POSIX_SOURCE
V7 ⇒ UNIX (and extern int errno)
COHERENT ⇒ UNIX ;  MINIX ⇒ UNIX
QNX ⇒ POSIX, CK_POSIX_SIG, (__16BIT__/CK_QNX16 or __32BIT__/CK_QNX32)
STRATUS ⇒ CK_ANSIC, NOSTAT
```

**The three "common-feature" rollups (most important for ckutio.c):**
```
SVORPOSIX     = ATTSV ∨ POSIX
SVR4ORPOSIX   = SVR4  ∨ POSIX
BSD44ORPOSIX  = BSD44 ∨ POSIX        (also forced on by IRIX64)
```

**Signal / select capability derivation:**
```
SELECT  defined if (not NOSELECT and) __linux__ ∨ SUNOS4 ∨ NEXT ∨ RTAIX ∨
        HPUX ∨ AIXRS ∨ BSD44 ∨ BSD4 ∨ OXOS ∨ BEBOX  (else from makefile)
CK_POSIX_SIG ⇐ QNX (else from makefile, e.g. linuxa)
SIG_V   ⇐ __DECC, apollo+__STDCPP__
SIG_I   ⇐ apollo(non-ANSI), A986
```

## 4. Dead vs. orphan flags

### 4.1 Tested in code but NOT defined by any *buildable* target

These have live `#ifdef` sites but cannot be reached from `macos*`, `linux*`,
`freebsd*`, `netbsd*`, or `openbsd*`. Most are reachable only from obsolete
makefile targets that no longer build (SVR4/AIX/HPUX/etc.), so they are
effectively dead on this tree even though the target text survives. A subset is
truly dead — never `-D` anywhere in the makefile:

Truly dead (no `-D` in the entire makefile, no compiler will predefine them on a
supported platform):
`MAC` (Classic Mac OS, 97 sites), `datageneral`, `Plan9`, `OSK`, `GEMDOS`,
`AMIGA`, `STRATUS`, `aegis`, `apollo`, `ultrix`, `sun`, `mips`, `pdp11`,
`SVR4ORPOSIX` is only *derived*, `MACOSX` is only *derived* (never `-D`).

Reachable only from obsolete (non-buildable) targets — dead in practice:
`SOLARIS*`, `SUNOS4*`, `HPUX*`, `AIX*`/`AIXRS`/`RTAIX`, `OSF*`, `IRIX*`,
`XENIX`, `SCO_OSR50x`, `CK_SCOV5`, `UNIXWARE`, `PTX`, `I386IX`, `OXOS`,
`COHERENT`, `MINIX*`, `QNX`, `NEXT`, `BSD43`, `EXCELAN`, `BELLV10`, `PROVX1`,
`BEBOX`/`BEOS`, `__bsdi__`/`BSDI`.

### 4.2 Defined by makefile but tested nowhere in the code

Cross-checking the makefile's OS `-D` flags against the code tests, the
OS-selector flags are all tested somewhere. The makefile flag with no
corresponding code test among OS selectors is:

- `-Dunix` (lowercase, line 7760) — passed only to the **wart** preprocessor
  build, not the Kermit sources; the code tests uppercase `UNIX`, not `unix`.
- `-Dopeni=xopeni` — not an OS selector; it is a symbol *rename* used by the HP-UX
  shared-library targets to avoid a libc clash. Listed here only to disqualify it.

(Everything else that looked like an orphan turned out to be a feature flag, not
an OS selector, and is therefore out of scope.)

### 4.3 Surprising / noteworthy findings

- **`MAC` ≠ `MACOSX`.** `MAC` is Classic Mac OS (MPW toolchain) with 97 `#ifdef`
  sites and is now completely dead; macOS is `MACOSX`/`MACOSX10`/`MACOSX103`.
- **No plain OS-name makefile flags.** There is no `-DFREEBSD`, `-DNETBSD`,
  `-DMACOSX`, or `-DSOLARIS` (without version digits) used as the primary
  selector; targets use versioned (`FREEBSD4`, `MACOSX10`) or family-base
  (`BSD44`) flags and let `ckcdeb.h` derive the rest.
- **NetBSD is selected purely as `BSD44`** — its target defines no `NETBSD`
  macro at all; only `__NetBSD__` (compiler-predefined) distinguishes it in code.
- **`UNIX` is almost never set on the command line** (one stray `-DUNIX`); it is
  derived from the OS family in `ckcdeb.h`. This is the single most-tested OS
  macro (293 sites).
- The live `linuxa` target's OS-relevant flags are exactly:
  `-DLINUX -DFNFLOAT -DCK_POSIX_SIG -DCK_NEWTERM -DLINUXFSSTND -DPOSIX`
  (plus TCP/string feature flags). `POSIX` then triggers the big POSIX/SVORPOSIX/
  BSD44ORPOSIX derivation cascade.
- The live `macos` target relies on `MACOSX10 ⇒ MACOSX ⇒ BSD44`, so macOS
  effectively compiles as a 4.4BSD variant; `BSD44 ⇒ SVR4`, so macOS also pulls
  in `SVR4`/`ATTSV`/`SVR3` despite being BSD-derived — a deliberate "4.4BSD is a
  SysV+POSIX+BSD mixture" design noted in `ckcdeb.h`.

### Counts

- Distinct OS-related macro names tested in code: ~110 (ifdef + ifndef forms).
- OS-related `-D` flags appearing anywhere in the makefile: ~60 distinct.
- OS selectors reachable from a *buildable* target on this machine: roughly
  `LINUX`, `POSIX`, `CK_POSIX_SIG`, `CK_NEWTERM`, `MACOSX10`, `MACOSX103`,
  `MACOSX`, `BSD44`, `FREEBSD4/8/9`, `OPENBSD`, plus everything those derive
  (`UNIX`, `ATTSV`, `SVR3`, `SVR4`, `SVORPOSIX`, `BSD44ORPOSIX`, `SELECT`,
  `DIRENT`, `NOFILEH`, `STERMIOX`, ...) — on the order of 25-30 effective macros.
- Effectively-dead OS macros (no buildable target, or never `-D`): the large
  majority — all of §4.1.
