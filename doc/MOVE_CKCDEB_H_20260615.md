# Pin `#include "ckcdeb.h"` first and protect it from clang-format `SortIncludes`

Date: 2026-06-15

## Motivation

`ckcdeb.h` is the master header included by every C-Kermit module. Its own header
comment states it **"should be included BEFORE any other C-Kermit header files"** ‚Äî
it establishes the platform/feature macros (`POSIX`, `BSD44`, `SVR4`, the debug
formats, the portability typedefs, etc.) that all the other `ck*.h` headers and the
`.c` sources depend on. Until now that ordering was enforced only by convention.

`.clang-format` deliberately keeps `SortIncludes: Enabled: false`, precisely because
turning it on would alphabetically sort sibling headers such as `ckcasc.h` **above**
`ckcdeb.h`, silently breaking the invariant.

This change makes `#include "ckcdeb.h"` the **first Kermit include** in every
`.c`/`.h` file and wraps it so `clang-format` can never reorder it, making it safe
to enable `SortIncludes` in the future. It is a behavior-preserving cleanup.

## Protection idiom

The single primary `ckcdeb.h` include in each file is wrapped with the project's
`/* */` comment style:

```c
/* clang-format off */
#include "ckcdeb.h"
/* clang-format on */
```

The `clang-format off` island is a hard barrier `SortIncludes` cannot reorder
across, so `ckcdeb.h` stays anchored as the top include while the remaining includes
stay sortable. A single-line wrap is sufficient (no need to freeze the whole include
block). With the current config (`SortIncludes: false`, `IncludeBlocks: Preserve`)
nothing reorders today; the wrap is future-proofing. `MainIncludeChar`/
`IncludeCategories` only apply when sorting is on, and the off-island is exempt.

Note ‚Äî `ckcmai.c`: `#define CKCMAI` legitimately appears *after* `ckcdeb.h`
(ckcdeb.h has `#ifndef CKCMAI` blocks that must compile); left untouched, only the
include was wrapped.

### Class B ‚Äî moved `ckcdeb.h` above a preceding Kermit include, then wrapped (2 files)

- `ckcfns.c` ‚Äî `#include "ckcasc.h"` (was line 29) preceded `ckcdeb.h` (line 30).
- `ckusig.c` ‚Äî `#include "ckcasc.h"` (was line 14) preceded `ckcdeb.h` (line 15).

`ckcdeb.h` is now first, immediately above `ckcasc.h`. **Safe on every platform:**
`ckcasc.h` is a fully self-contained header (its own `#ifndef CKCASC_H` guard, only
ASCII-mnemonic `#define`s, no `#include`s, and no reference to any `ckcdeb.h`
symbol), so the two headers are order-independent.

### Class C ‚Äî load-bearing pre-include context: pinned in place, **not** hoisted (5 cases)

`ckcdeb.h` stays below the noted "floor" and was only wrapped where it already sits:

| File | Floor `ckcdeb.h` must stay below | Reason |
|------|----------------------------------|--------|
| `ckwart.c` | `#define CKWART_C` (line 1) | `ckcdeb.h:733` and `ckclib.h:73` test `#ifndef CKWART_C`. Kept at line 6 (after the `<stdio/stdlib/string.h>` trio ‚Äî not hoisted above system headers). |
| `ckupty.c` | `#ifndef NO_PTY_XOPEN_SOURCE` / `#define _XOPEN_SOURCE 500` + `<stdlib.h>` | `_XOPEN_SOURCE` is a feature-test macro that must precede system headers, which `ckcdeb.h` pulls in. |
| `ckctel.c` | `#define CKCTEL_C` + `#ifndef NONET` / `#ifndef NOTCPIP` | Hoisting out of the guards would change when `ckcdeb.h` compiles in a `NONET` build. Pinned inside the guards (already first include). |
| `ckcnet.c` | `#ifdef NORLOGIN`/`#undef RLOGCODE` + `#define CKCNET_C` | The undef and self-id define stay above the include. |
| `ckcpro.w` | line-41 include is inside `#ifndef NOXFER`; the second copy is in the `#else /* NOXFER */` branch | Both are legitimate branch-primary includes (see below). |

### Generated file ‚Äî `ckcpro.c` / `ckcpro.w`

`ckcpro.c` is generated by `wart`; **`ckcpro.w` was edited, never `ckcpro.c`.** `wart`
copies the pre-`%%` prologue (`putc`) and the post-rules region (`copyrest`)
verbatim, so the wrapper comments pass through. Both `ckcdeb.h` includes were wrapped
(the `#ifndef NOXFER` branch and the `#else /* NOXFER */` branch), then regenerated:

```sh
make wart && make ckcpro.c
```

Confirmed present in regenerated `ckcpro.c` at lines 49‚Äì51 and 3779‚Äì3781.

## Explicitly left untouched

- **Redundant second `ckcdeb.h` includes** at `ckucmd.c:200` and `ckwart.c:53` ‚Äî
  harmless header-guarded no-ops (`#ifndef CKCDEB_H`), not "first" includes. Not
  wrapped, not removed (removal would be unrelated churn).
- **`.clang-format`** ‚Äî `SortIncludes` stays disabled; the wrappers merely make
  enabling it safe later.

## Verification

1. **Wrap audit:** every `.c`/`.h` (and regenerated `ckcpro.c`) has its primary
   `#include "ckcdeb.h"` bracketed by `/* clang-format off */` ‚Ä¶ `/* clang-format on */`,
   and `ckcdeb.h` is the first *Kermit* include everywhere. 32 marker pairs across 31
   files (`ckcpro.w` has 2).
2. **`make check`:** SUCCESS.
3. **Whole-binary byte-identical proof (Linux path):** with **both** trees built
   fully clean under a fixed `SOURCE_DATE_EPOCH`
   (`make clean && make wart && make ckcpro.c && make linux`), the modified tree's
   `wermit` is **`cmp`-identical** to a pristine `git archive HEAD` build. The
   changes are comments plus a reorder of two order-independent symbol-only headers,
   so there is zero codegen impact. Cross-checked at the object level: **all 29
   `*.o` are byte-identical** (`ckcpro.o` included).
   - Caveat learned: a *non-clean* baseline (`rm wermit && make linux` without
     `make clean`) relinks stale `.o` carrying an earlier `__DATE__`/`__TIME__`,
     which perturbs the GNU build-id and produces a spurious early-byte diff
     (observed at byte 865). Always compare two **fully clean**, fixed-epoch builds.
4. **macOS/BSD paths:** unaffected by reasoning ‚Äî the only structural change off the
   Linux path is the Class-B header reorder, and `ckcasc.h` is self-contained and
   order-independent (above); the Class-C floors preserve `_XOPEN_SOURCE`, `CKWART_C`,
   the `NONET`/`NOXFER` guards, and the `RLOGCODE` undef on every target. The wrapper
   lines are comments and inert on all `#ifdef` paths.
