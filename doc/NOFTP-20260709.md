# Codebase Simplification Report — Removal of the built-in FTP client (`NOFTP` always defined)

**Date:** 2026-07-09
**Tree:** C-Kermit 10.0 Beta.12 (`/home/kenji/src/simplified-c-kermit`)
**Scope:** Remove the built-in FTP client subsystem altogether by treating the compile
flag **`NOFTP` as permanently defined**. The whole `ckcftp.c` FTP client and its private
header `ckcftp.h` are deleted; every `#ifndef NOFTP` / `#ifdef NEWFTP` / `#ifdef SYSFTP`
feature block scattered through the interactive UI is resolved away. Non-FTP code — the
Kermit protocol engine, Telnet, serial/tty I/O, TCP networking, CONNECT — is untouched;
`TCPSOCKET`/`NETCONN` builds keep everything except FTP.

Build verification on Linux (this host): **success** — `make linux` exits 0, **zero
compiler warnings**, binary `wermit` links cleanly without `ckcftp`. The version banner is
unchanged (`C-Kermit 10.0.416 Beta.12, 2025/03/22`); `SHOW FEATURES` now lists **`NOFTP`**
(and no longer `NEWFTP`), confirming FTP is cleanly absent rather than merely dormant.

## Method

1. **Analysis** was performed by a `c-expert` subagent, which produced the macro-logic
   proof (below), the complete guard-region inventory, the dangling-symbol audit, and the
   hazard list. It validated the mechanical transform by running `unifdef` on file copies,
   reviewing every diff hunk, and building + runtime-testing a full scratch copy of the tree
   before any change was made to the real repository.
2. **Whole-file deletion** of `ckcftp.c` (14,084 lines) and `ckcftp.h` (45 lines, already
   `#include`d by nothing in the tree — a pre-existing orphan).
3. **Conditional-block removal** with `/usr/bin/unifdef -DNOFTP -UNEWFTP -USYSFTP -m`
   (in-place) over the 12 `.c`/`.h` files bearing FTP guards, with per-file exit-status
   checking (all exit 1 = modified; no parse errors).
4. **Hand edits** to `ckcdeb.h` (the macro machinery — `unifdef` cannot cleanly collapse the
   definition cascades) and to five call-site hazards `unifdef` leaves behind.
5. **`clang-format`** re-normalization of all 14 touched `.c`/`.h` files (the guard removals
   un-nest code and shift comment alignment), then the project format-stability gate
   (`clang-format FILE | diff FILE -` empty) confirmed clean on every one.

## Macro logic — why forcing `NOFTP` also kills `NEWFTP` and `SYSFTP`

`NEWFTP` ("built-in FTP client", the default on this tree) and `SYSFTP` ("shell out to the
system `ftp` program") are `#define`d in **exactly one place** — the arbitration block at the
old `ckcdeb.h:1401–1426` — which is entirely gated `#ifndef NOFTP`. No makefile target passes
`-DNEWFTP`/`-DSYSFTP`, and nothing else defines them. Therefore defining `NOFTP` makes that
block's `#ifndef NOFTP` false, so `NEWFTP` and `SYSFTP` can never become defined. This logic
has **no platform conditions around it**, so the result is platform-independent.

Proved with `gcc -E` on a probe that `#include`s the **post-edit** `ckcdeb.h`, for all three
live target families:

| Target flags | `NOFTP` | `NEWFTP` | `SYSFTP` | preproc errors |
|---|---|---|---|---|
| `-DLINUX -DCK_POSIX_SIG -DTCPSOCKET -DPOSIX` | defined | undef | undef | 0 |
| `-DMACOSX10 -DMACOSX103 -DTCPSOCKET -DCKHTTP` | defined | undef | undef | 0 |
| `-DBSD44 -DTCPSOCKET -DOPENBSD` | defined | undef | undef | 0 |

Consequently the transform resolves three symbols at once: `NOFTP` **defined**, `NEWFTP`
**undefined**, `SYSFTP` **undefined** — `#ifndef NOFTP`/`#ifdef NEWFTP`/`#ifdef SYSFTP`
bodies are deleted; `#ifdef NOFTP`/`#ifndef NEWFTP`/`#ifndef SYSFTP` and `#else /* NOFTP */`
bodies are kept and un-guarded.

## What was removed

### Whole-file deletions
- **`ckcftp.c`** — the entire FTP client (14,084 lines), which was already wrapped
  `#ifndef NOFTP` → `#ifndef SYSFTP` → `#ifdef TCPSOCKET` and compiled to nothing under
  `NOFTP`. `git rm`, matching the campaign precedent (`ckcsym.h`, `ckusig.h`, `ckcmdb.c`, …
  were deleted outright, not stubbed — see `doc/BASE_20260616.md` §3).
- **`ckcftp.h`** — its 45-line private prototype header, `#include`d by no other module.

### `ckcdeb.h` — macro-machinery collapse (hand-edited)
- The old **arbitration block** (1401–1426) and the dead **`#ifdef NEWFTP` prototype block**
  (`ftpisconnected`/`ftpisloggedin`/`ftpissecure`, 3678–3682) were removed.
- The **8 scattered feature-implied `#ifndef NOFTP #define NOFTP #endif` trios** (under
  `NODEPRECATED`, `NOICP`, `NOXFER`, `IKSDONLY`, `NOLOCAL`, `NONET`, `NOFRILLS`, `NOTCPIP`)
  were deleted, along with the now-moot `#undef NEWFTP`/`#undef SYSFTP` lines in the
  `NOTCPIP` block. Only the FTP lines were touched; each block's other feature `#define`s
  (`NOTELNET`, `NOCURSES`, `NO_COMPORT`, `NOHTTP`, `NOBROWSER`, `TCPSOCKET` undef, …) were
  left intact.
- A single **unconditional definition** was added near the top of the file (after the
  `NOWTMP`/`NOSYSLOG` "deprecated, off by default" block it mirrors), with a comment noting
  that FTP has been removed, that there is deliberately **no `DOFTP` escape hatch**, and
  pointing here.

### `unifdef` pass — 12 files, all pure deletions
`ckcmai.c` (1), `ckucns.c` (2), `ckuus2.c` (8+15 kept), `ckuus3.c` (12), `ckuus4.c` (8),
`ckuus5.c` (9+3 kept), `ckuus6.c` (1), `ckuus7.c` (6), `ckuusr.c` (18+2 kept), `ckuusr.h` (2),
`ckuusx.c` (14), `ckuusy.c` (7). These are the FTP command tables, the `GET`/`PUT`/`REMOTE`
FTP dispatch, the `SET FTP`/`SHOW FTP` machinery, the CONNECT-time and transfer-display FTP
branches, and the `ftp://`/`ftps://` command-line personality. Every `#ifdef NEWFTP … else
#endif { body }` idiom (a common C-Kermit pattern) correctly kept its non-FTP `else` body,
un-nested — e.g. `ckucns.c`'s "you must SET LINE or SET HOST first", `ckuus3.c`'s
`dologshow()` (`ftp` is now always 0) and the `"ftp", 1, 0` feature-table entry.

### Manual call-site follow-ups (hazards `unifdef` leaves behind)
- **`SHOW FTP` (behavior change, intended).** The `SHOFTP` keytab entry lived under
  `#ifdef BROWSER` — *not* gated by `NOFTP` — while its handler (`case SHOFTP:`) was gated
  `#ifndef NOFTP`. Left mechanical, `SHOW FTP` would have parsed but fallen through to
  "Nothing to show…". The keytab entry (`ckuusr.c`) was removed, so **`SHOW FTP` is now an
  unrecognized keyword**, consistent with the plain `FTP` command's `XXNOTAV` treatment.
- **Orphaned `case XXFTP:` label** in `ckuus2.c` (the `case` sat outside the FTP guards, its
  bodies inside): the label was removed so it can't fall through into the `PRINT` help.
  `XXFTP` is provably unreachable — its only producers were the deleted `ftp` keytab entries.
- **Dangling prototypes** removed for a clean tree: the 6 orphaned `ckcfnp.h` prototypes
  (`doftpget`, `doftprmt`, `doftptyp`, `ftpopen`, `shoftp`, `doftpglobaltype`) and the
  `static int xx_ftp(...)` forward declaration in `ckuusy.c` (the only one with real
  `-Wall`/`-Wextra` exposure, being `static`-declared-but-undefined).
- **Unused local** `int t` in `ckuus7.c`'s `clskconnx()` (it only saved/restored `ftpget`)
  reduced to `int rc`.

### `makefile`
`ckcftp.$(EXT)` removed from all 5 link-object lists (`wart`/`wermit`/`xermit` targets) and
the `ckcftp.$(EXT):` build rule deleted. No target referenced FTP via `KFLAGS`.

## Behavior changes

All removals are behavior-preserving **dead/unreachable-code elimination** except two
user-visible dispositions, both deliberate and both the correct outcome of "no FTP":

- `FTP`, `HELP FTP`, and (the `TCPSOCKET` help entry) `REMOTE`-style FTP commands report
  *"…not configured in this version of Kermit."* (`XXNOTAV`), exactly as before for a
  `NOFTP` build.
- `SHOW FTP` is no longer a recognized `SHOW` sub-keyword (see hazard above).

`SHOW FEATURES` gains `NOFTP` and loses `NEWFTP`. The `NOFIREWALL` cascade that the kept
`#ifndef NEWFTP` branch (`ckuus2.c`) re-activates is a **non-issue on the live targets**:
`CKHTTP` is unconditionally defined on Linux/BSD, so its inner `#ifndef CKHTTP` never fires —
`HELP FIREWALL` still prints the full proxy-support text (verified at runtime).

## Verification

- **Build:** `make clean && make linux` → exit 0, **warning-clean** (checked excluding the
  benign `USE_STRERROR`/`FOOERROR` substring matches); `wermit` links with no `ckcftp` object.
- **Portability:** `gcc -E` proof above shows the `ckcdeb.h` collapse yields
  `NOFTP`/`¬NEWFTP`/`¬SYSFTP` with zero preprocessor errors on the linux, macOS
  (`MACOSX10`), and modern-BSD (`BSD44`) define sets. *(A full `make macos` on daimac is a
  recommended follow-up but was not run for this doc; the header proof + the whole-tree
  dangling-symbol audit + the linux build cover the risk.)*
- **Format stability:** all 14 touched `.c`/`.h` files pass `clang-format FILE | diff FILE -`
  (empty). The `clang-format` re-flow did not change the binary size, confirming it is
  whitespace/brace-only.
- **Runtime smoke tests** (`./wermit -Y -q`):
  - `ftp` → "Sorry, "ftp" not configured in this version of Kermit."
  - `help ftp` → "This command is not configured in this version of Kermit."
  - `show ftp` → unrecognized keyword (clean "command error"), **not** "Nothing to show".
  - `help firewall` → full firewall/proxy help intact (non-FTP feature preserved).
  - `show features` → `NOFTP` present; `NEWFTP`/`SYSFTP` absent.
  - `PAUSE 2` → 2.006 s wall time (SIGALRM `longjmp` timer healthy — general signal sanity).

## Residuals / deliberately kept

- The command-code `#define`s **`XXFTP`** (`ckuusr.h`) and **`SHOFTP`** (`ckuusr.h`) remain
  as harmless unused constants in their numbered tables; removing numbered command codes
  mid-table is cosmetic churn with no benefit, so they were left.
- `ckuus4.c`'s `howcalled = I_AM_FTP;` (under `#ifndef NOURL`) survives, but nothing checks
  `howcalled == I_AM_FTP` anymore (both check-sites were deleted), so it is an inert tag
  reachable only via an `ftp://` first-argument in a `NOURL`-off build. Optional cleanup.
- No `DOFTP` re-enable switch was added: FTP removal is intended to be permanent for this
  tree. Re-enabling would require restoring `ckcftp.c` and its command tables.
