## 1. Feature removals — FTP, RLOGIN, XMODEM/YMODEM/ZMODEM clients

Three built-in client features were removed by hard-wiring the compile flag that already disabled
each of them, following the same `unifdef`-plus-hand-fix method established in the 20260709 range's
security work. Combined (`db488c9^..5449df7`, the three removal commits): **30 files changed,
+798/−17,248** (net **−16,450** lines) — the bulk of this range's real (non-churn) deletion.

| Removal | Commit | Flag hard-wired | Files | Diff | `wermit` delta | Report |
|---|---|---|---|---|---|---|
| FTP client | `db488c9` | `NOFTP` (implies `¬NEWFTP`, `¬SYSFTP`) | `ckcftp.c`/`.h` deleted, 12 more `unifdef`-ed, 5 hand-edited | 18 files, +260/−15,328 | −194,888 bytes | `doc/NOFTP-20260709.md` |
| RLOGIN client | `fa245f6` | `NODEPRECATED` → `NOTELNET`+`NORLOGIN` hard-wired, `RLOGCODE` killed | 14 `unifdef`-ed, 7 hand-edited | 19 files, +312/−1,092 | −4,448 bytes | `doc/NODEPRECATED-20260709.md` |
| XMODEM/YMODEM/ZMODEM | `5449df7` | `NOCKXYZ` → `CK_XYZ` killed | 16 files, 15 `unifdef`-ed | 17 files, +224/−828 | −9,232 bytes | `doc/NOCKXYZ-20260709.md` |

Two README commits (`7acd122`, `269f2a5`) record these under "Achievements" the same session.

**What deliberately remains untouched in all three:** the Kermit protocol engine, Telnet (the
`TELNET` command and `ckctel.c` engine are fully functional — the RLOGIN removal explicitly did
**not** touch Telnet, correcting `NODEPRECATED`'s own inaccurate "no Telnet client" `SHOW FEATURES`
line rather than perpetuating it), serial/tty I/O, TCP networking, CONNECT, and IKSD (which needs
Telnet negotiation and would otherwise have been dragged out with a literal reading of
`NODEPRECATED`). `REDIRECT` and the generic external-command machinery (`ttruncmd()`,
`exp_handler`/`exp_timo`) that XYZMODEM used are explicitly kept — they're shared with other
features, not XYZ-specific.

**User-visible effects:**

- `FTP`, `HELP FTP`, FTP-style `REMOTE` commands → *"not configured in this version of Kermit."*
  `SHOW FTP` is now an **unrecognized keyword** (its keytab entry sat outside the `NOFTP` guard and
  had to be removed by hand to avoid a "falls through to Nothing to show" mismatch).
- `rlogin` command → *"?Sorry, RLOGIN is not configured…"* (unchanged from a `-DNODEPRECATED`
  build); `help rlogin` → "help not available"; `set host <host> /rlogin` falls through to a
  service-name lookup failure; `set host <host> 513` now opens a generic (untagged) TCP connection
  instead of the rlogin handshake; a binary invoked/symlinked as `rlogin` gets generic Kermit
  personality instead of dead rlogin-flavored usage text (a latent inconsistency fixed as part of
  the removal, not introduced by it).
- `SET PROTOCOL` → `kermit` only; `SET TRANSFER PROTOCOL` and `SET EXTERNAL-PROTOCOL` keywords are
  gone entirely; `SHOW PROTOCOL`/`HELP SET PROTOCOL` shrink to the Kermit-only text; CONNECT-mode
  autodownload no longer recognizes ZMODEM `rz` start sequences (Kermit autodownload untouched);
  `SEND /PROTOCOL:` remains parseable but hidden (`CM_INV`) and kermit-only.
- `SHOW FEATURES` gains `NOFTP`/`NOTELNET`/`NORLOGIN`/`NOCKXYZ`("No built-in XYZMODEM protocols")
  and loses `NEWFTP`/`RLOGCODE`/"External XYZMODEM protocol support".

Each removal's own report documents dangling-reference hazards found and fixed by hand beyond the
mechanical `unifdef` pass (e.g. FTP's orphaned `case XXFTP:` label, RLOGIN's unguarded numeric
service-513 case and `argv[0]`-personality clause, XYZMODEM's orphaned `zmdlok` declarations) — see
the individual reports for the full list; none is repeated here.

## 2. New makefile targets — `macos-notcp` / `macos-nonet`

`ee6e087` adds macOS equivalents of the existing `linux-notcp`/`linux-nonet` targets (thin `KFLAGS`
wrappers re-invoking `macos`), plus a fix: the `#ifndef NETCONN` dummy-function block in
`ckcnet.c` (compiled only under `NONET`, never by `make linux`/`make macos`) still had three K&R
definitions the 2026-06 K&R sweep had missed — they produced `-Wdeprecated-non-prototype` warnings
under macOS clang and were converted to ANSI form. `6870b23`/`388476b` document both the Linux and
new macOS directives in the README. Combined (`269f2a5..388476b`): **4 files changed, +102/−13**.

One non-obvious wrinkle the change report works out: `macos`'s target hard-codes `-DCKHTTP` in its
own `CFLAGS`, and nothing else in `ckcdeb.h` undefines a command-line `-DCKHTTP` the way `NONET`/
`NOTCPIP` already neutralize `TCPSOCKET` — so both new targets add an explicit `-UCKHTTP` to keep
`SHOW FEATURES` honest (otherwise it would list `CKHTTP` alongside `NOHTTP`, contradictory but
harmless). Verified on real macOS hardware (`daimac`, arm64, Apple clang 21.0.0): all three targets
(`macos`, `macos-notcp`, `macos-nonet`) build warning-clean; `SHOW FEATURES`/`SHOW NET` confirm the
expected feature set at each level. Report: `doc/nonet-macos-20260709.md`.

## 3. Preprocessor modernization — the `#ifdef`-nesting campaign

Triggered by a concrete performance complaint: `clang-format` on this tree's smallest file
(`ckcdeb.h`, 3,705 lines) took **~82 seconds**, ~70–83× longer than files 6× its size, because of
sustained runs of deeply-nested `#ifdef`/`#else` cascades standing in for `#elif`. A from-scratch
audit (`doc/IFDEF-NESTING-20260710.md`) built an accurate comment/string-aware nesting-depth
scanner (a naive line-anchored regex mis-scored one file, `ckuus3.c`, due to a comment containing
the literal text `#ifdef HPUX10..#endif.`), measured every one of the 44 in-scope files, and
proposed four fixes in priority order. All four were applied and are committed
(`doc/IFDEF-UNROLL-20260710.md` is the running log); combined source diff
(`ce1a521^..008c216`, `*.c`/`*.h` only): **9 files changed, +63/−307** (net **−244 lines**).

| Phase | Commit | What | Depth / timing result |
|---|---|---|---|
| 1 | `ce1a521` | `ckcdeb.h`'s 11-step `BPS_*`/`MAX_SPD` speed ladder → one `#elif` chain | File max 13→9 (a second, previously-hidden depth-9 cascade, `USE_LSTAT`, was exposed); `clang-format` ~82 s → ~30.6 s (2.68×) |
| 2 | `b5cdee6` | `#elif`-chain rewrites in `ckuusx.c` (network-transport ladder), `ckcmai.c` (`nettype`), `ckucmd.c` (libc-internals ladder), plus `ckcdeb.h`'s `USE_LSTAT`/`CK_64BIT` — **but two more `ckcdeb.h` cascades (`openpty()` ×2) were tried, found to regress `clang-format` time ~4.7× (super-additive, tied to `#include` directives inside the arms), and reverted** | `ckcdeb.h` final `clang-format` time ~23.4 s (faster than Phase 1, since `USE_LSTAT`/`CK_64BIT` are free-to-negative) |
| 3 | `5c40ec2` | Deleted the three `#ifdef NOTUSED` dead blocks (`NOTUSED` is `#define`d nowhere in the tree) — `ckcnet.c`'s 147-line `tcpsocket_open()` (which doesn't even parse — a missing close-paren, proof it hasn't compiled in years), `ckuxla.c`'s dead `yasl1[]` table, `ckufio.c`'s dead `zkermini()` | 172 confirmed-dead lines removed; a scanner off-by-one line-numbering bug (multi-line trailing comments) was found and fixed along the way |
| 4 | `008c216` | Collapsed genuine AND-nests with no `#else` in the span: `ckutio.c`'s 4-level `_IBMR2`/`TIOCMBIS`/`TIOCMBIC`/`TIOCM_DTR` chain, and **both** occurrences of `ckctel.c`'s `IKSDONLY`/`CK_AUTODL` pair (the audit had only named one; re-scanning per its own instruction found a second, tied-depth occurrence) | `ckcnet.c`'s six `NOTCPOPTS`/`SOL_SOCKET`/`SO_*` triples were **correctly skipped** — re-reading found real `#else` fallback code at both levels, not a clean AND-nest as the audit's Pattern-2 characterization claimed |

**Tree-wide max nesting depth: 13 → 9.** 6 files improved (`ckcdeb.h` 13→9, `ckutio.c` 9→8,
`ckuusx.c` 8→6, `ckctel.c` 7→6, `ckcmai.c` 6→5, `ckucmd.c` 7→5); 38 unchanged.

The campaign's most important finding is a **correction to its own audit's central thesis**: two
distinct `clang-format` regression mechanisms were discovered by measuring (not assuming)
before/after timing on every hunk, both reproducible and neither reducible to "nesting depth" or
"line count" alone — (a) `#include` directives inside collapsed cascade arms cause a
**super-additive** slowdown once more than one such region exists in a file (the reverted
`openpty()` cascades: +41 s each alone, +100 s together vs. a +82 s additive prediction), and (b) a
merged `#if` condition long enough to force a `clang-format` backslash line-wrap can itself cost
measurably more on every subsequent format, independent of the merge (`ckutio.c`'s `_IBMR2` chain:
neutral before formatting, +15% after). One optional `ckutio.c` merge (`HUP_POSIX`/`BSD44ORPOSIX`,
+43%) was tried and reverted for this reason; the kept `_IBMR2` merge's smaller +15% cost was
disclosed and accepted since the merge is independently correct. Every kept and reverted hunk was
re-verified at Level 1 (`SOURCE_DATE_EPOCH` byte-identical rebuild — the campaign's `wermit` md5
never changed from `eee3a8218e33b4e6a495f1fe008a43c5` across all four phases) and Level 3
(`gcc -E -P` token-stream equivalence, 34+ combinations in Phase 4 alone).

Left deliberately alone, flagged as ready-to-apply future opportunities: the two reverted
`openpty()` cascades in `ckcdeb.h`; a `<time.h>`-selector cascade in `ckcnet.c` exposed by Phase 3
(`NOHTTP`/`TIMEH`/`SYSTIMEH`/`SYSTIMEBH`/`POSIX`/`CLIX`, plus a self-negating `#ifdef SYSTIMEH`
nested inside `#ifndef SYSTIMEH` worth a human look); a UUCP lock-directory cascade in `ckutio.c`
exposed by Phase 4; a pre-verified, in-scope-but-unrequested `ckcdeb.h` AND-nest around `SSHCMD`.
None of the `V7MIN`-gated `NOICP`/`NONET`/`NOLOCAL`/`NOXFER` whole-section wrappers were touched —
they gate a real, currently-invocable (if unwired to any `make` target) minimum-size build mode.

## 4. C++-style comment conversion

`2c63c07` first taught the `wart` preprocessor (`ckwart.c`) to skip `//` line comments via a new
`rdcmns()` function, alongside the existing `/* */` handler `rdcmnt()`. `947ce2a` and `2d08b8a`
then converted `ckcpro.w` and `ckwart.c` itself to `//` style. Finally `3ae6ed7` converted **every
other `.c`/`.h` file in the tree** (42 files) the same way — the largest single commit in this
range: **+64,488/−66,667**. Combined whole-campaign source diff (`2c63c07^..3ae6ed7`): **44 files
changed, +65,957/−68,436** (net **−2,479** lines — mostly disappearing standalone `/*`/`*/`
delimiter lines; every file shrank except `ckcsig.h`, +2, where dense single-line multi-clause
comments unpacked into one `//` per clause).

Verified functionally equivalent at the strongest available level
(`doc/CPP-STYLE-COMMENTS-20260711.md`, comparing `main` at `008c216` against the branch tip
`3ae6ed7`): **all executable code in `wermit` is byte-identical** — 26 of 28 object files
byte-identical outright, and the linked binary differs only in the (necessarily-differing)
GNU build-id and one intentional `.rodata` string (the wart version banner, 2.17→2.18). A
purpose-built Python phase-2/phase-3-faithful comment stripper confirmed **43 of 43** comment-only
files (42 `.c`/`.h` + `ckcpro.w`) are token-identical after comment removal, with **zero**
string/character-literal drift across all 620+ literals checked. `ckwart.c`'s one real functional
change (the `rdcmns()` addition) was reviewed line-by-line and found correct — it operates only on
the region of `ckcpro.w` between the two `%%` markers, where a bare `/` was already a fatal error,
so classic edge cases (`//` inside a string, `/` as division) are structurally unreachable to it.

A genuine hazard class was scanned for and found inert: **9 lines across 6 files** end a new `//`
comment with a trailing backslash, which (per C's phase-2 line-splicing) folds the *next* physical
line into the comment — every one of the 9 was confirmed to fold in another comment or blank line,
never live code, but they are flagged as **future-edit tripwires** (a later edit adding real code
right after one, or stripping a continuation line's leading `//`, would silently comment code out).

**One recommendation from that report was not carried out before merge**: the report found the
branch regressed the project's `clang-format`-stability gate on 11 of 44 files (`main` was 0/44
before the conversion) and recommended a single `clang-format -i` pass over the 10 convergent
ones before merging. That pass was never applied within this range — `main` at `6d386d9` still
had all 11 files (`ckcfn2.c`, `ckcfns.c`, `ckucon.c`, `ckudia.c`, `ckufio.c`, `ckutio.c`,
`ckuus3.c`, `ckuus5.c`, `ckuus6.c`, `ckuus7.c`, `ckuusy.c`) failing
`clang-format FILE | diff FILE -`. All were cosmetic (trailing-`//` column alignment, single
line-break placement) with zero functional risk per the token-identity proof. **Closed
immediately after this range by commit `73493da`** (same day, while preparing this document):
`clang-format -i` applied to the 10 convergent files (`ckutio.c` needed two iterations to
converge), verified by per-file `gcc -E -P` token-stream identity and a fixed-epoch rebuild
**byte-identical** to the §5/§9 baseline (same SHA-256). `ckufio.c` remains the known,
separately-documented `ReflowComments` oscillation — see §8 (since closed, same day — see §11).

## 5. `ckcdeb.h` reduction

`3d183bd` hard-wires eight more flags permanently on (`NOWTMP`, `NOSYSLOG`, `NOFTP`, `NOTELNET`,
`NORLOGIN`, `CK_ANSILIBS`, `ZCHKPID`, `NOCKXYZ`), removing their now-redundant `#ifndef` guards and
the `DOWTMP`/`DOSYSLOG` "explicitly re-enable" escape hatches; deletes three fully dead, referenced-
nowhere macros (`CKNTSIG` — a Windows-NT signal hack, `NO_FTP_AUTH`, `VMS64`); restructures the
`CKMAXNAM` cascade to use `#else` (a form the project's *own* now-lifted old preprocessor-portability
restriction used to forbid); and fixes several swapped `#endif` trailing-comment labels
(`CKREALPATH`/`CKSYMLINK`). Single file, **+45/−129** (net **−84** lines; `ckcdeb.h`: 3,554 → 3,470).

Verified functionally equivalent (`doc/CKCDEB-REDUCTION-20260711.md`): **byte-identical `wermit`**
(same SHA-256, same 2,655,672-byte size) and all 28 object files, plus `gcc -E -dM` macro-definition
dumps of `ckcdeb.h` across **five** live platform-flag families (Linux, macOS, generic/FreeBSD,
OpenBSD, NetBSD — each confirmed to correctly cascade `MACOSX10`/`BSD44` → `SVR4`/`SVR3`/`ATTSV` →
`UNIX` per the `CLAUDE.md` warning) showing the *same two* differences in every family: a cosmetic
`CK_SLEEPINT 250` → `(250)` reparenthesization (zero consumers of the macro anywhere in the tree)
and the `NO_FTP_AUTH` removal. No other touched macro — including all eight newly-hard-wired
flags — differs on any platform family. The `DOWTMP`/`DOSYSLOG` escape hatches, while genuinely
functional in the old header (confirmed by probing `-DDOWTMP -DDOSYSLOG` directly), were never
set by any makefile target, so removing them is a no-op for every shipped build; a related,
independent, pre-existing bug was also found and noted (the `macos` target's own comment claiming
`KFLAGS=-UNOWTMP` re-enables Wtmp logging was already non-functional before this diff, since
nothing predefines `NOWTMP` via `-D` for a bare `-U` to cancel — not introduced or worsened here).

- **Gone:** `FTP` (and `SHOW FTP`), `rlogin`, and `SET PROTOCOL {x,y,z}modem` / `SET

  `#ifdef`-flattening campaign, the comment conversion, or the `ckcdeb.h` reduction — all three
  are proven byte-identical-binary changes (§3–§5, §9).
- **Cosmetic-only, in-source:** every comment in the tree is now `//`-style rather than `/* */`

### 11.2 `c456225` — `ckufio.c` `ReflowComments` oscillation, closed

function` on `switch (x) {`, `// Set the creation date of the file` on `case 0:`, `// And

already identified as the tree's one remaining `clang-format`-stability failure (a `ReflowComments`
oscillation that never converges). The fix moves the three trailing comments to their own
standalone lines above `switch (x) {` and re-indents the `CK_PERMS` block comment to the `case`
body's actual 4-space level instead of the hardcoded column. Zero code tokens move, add, or
change — confirmed two ways:
