## Phase 1 — Critical (`7cf795c`)

- **V-3 — `xunchar()` clamp.** `ckcker.h:653`. Redefined `xunchar(ch)` from `((ch)-SP)&0xFF`
  (which wraps control bytes 0x00–0x1F to 225–255) to
  `(((ch) < SP || (ch) > (SP+94)) ? 0 : (((ch)-SP)&0xFF))`, clamping any out-of-range encoded
  byte to 0 ("field absent"). Central fix for the whole packet-decode path. Verified the
  `#ifndef xunchar` fallback in `ckutio.c` is inert (`ckcker.h` is included ahead of it) and that
  no arithmetic-combining call site relied on the wraparound.
- **V-1 — pre-auth REMOTE LOGIN overflow.** `ckcpro.w` `srv_login()` (~2611). Added
  `RESUME; return(-1);` inside the `if (len > LOGINLEN)` branch so an over-long username aborts
  instead of falling through into the 33-byte copy. `ckcpro.c` regenerated via `make wart && make ckcpro.c`.
- **V-5 — pre-auth Telnet NEW-ENVIRON reply overflow.** `ckctel.c` `tn_snenv()` pass 2 (~4083).
  Wrapped the `default: varname[j++] = sb[i];` write in `if (j < 16)`, mirroring pass 1 and
  `tn_rnenv()`.
- **V-18 — off-by-one past `varname`.** `ckctel.c:3714`. `char varname[16]` → `char varname[17]`
  (the `varname[j]='\0'` terminator can legitimately write index 16). Applied together with V-5.
- **V-2 — unbounded `putsrv()` heap write.** `ckcfns.c` `putsrv()` (~414). Gated the store on
  `if (srvptr < srvcmd + srvcmdlen)`; the terminating NUL still lands inside the buffer.
- **V-4 — unbounded Attributes reply overflow.** `ckcfn3.c`. Added `#define RPBUFL 20`,
  `static char rpbuf[RPBUFL+1]`, and a bounded `RPBUFPUT(ch)` macro (`ckcfn3.c:1339`); converted
  the 10 documented `*rp++ = c;` append sites to `RPBUFPUT(c)`.
  **Deviation / extra:** two further unbounded writes not in the proposal's 10-site list —
  `*rp++ = '1';` and `*rp++ = '!';` in the length-refusal branches (`ckcfn3.c:1793`, `1803`) —
  were also converted to `RPBUFPUT('1')` / `RPBUFPUT('!')`. Without them, when a file is
  *refused* (`retcode == -1`, so the reset at line 1828 is skipped) `rp` could sit one past the
  cap and the unconditional terminator at line 1830 would write one byte past `rpbuf`. This closes
  a residual single-byte overflow the exact documented patch left open.
- **V-7 — FTP PASV `227` parser overflow.** `ckcftp.c` (~9707). Guarded the `pasv[]` copy with
  `if (pt < &pasv[sizeof(pasv) - 1])`, matching the sibling `ftp_reply_str` bound.
- **V-8 — server-mode GET filename array overflow.** `ckuusx.c` `fnparse()` (~2335). Guarded
  `msfiles[r] = p; r++;` with `if (r < MSENDMAX - 1)` (headroom for the terminating `""` entry).
- **V-6 — path traversal via unstripped `..`.** `ckufio.c`. Added a lexical `zhasdotdot()` helper
  that flags a `..` path *component* (not `..foo`) and applied it in `nzrtol()`, falling back to
  `zstrip()`'s bare-filename behavior on a hit.
  **Deviation:** the proposal placed the helper "after `zstrip()`", but in the actual source
  `nzrtol()` *precedes* `zstrip()`. Since the tree builds as C23 (an undeclared function is a hard
  compile error), the helper was placed immediately **before** `nzrtol()` (its only caller,
  `ckufio.c:2438`) instead. Body and guard are otherwise as proposed.

## Phase 2 — High (`d85fd58`)

- **V-9 — CONNECT-mode `ecbuf[10]` overflow (ASan-confirmed).** Bounded the escape-collection loop
  with `if (ecbp < ecbuf + sizeof(ecbuf) - 1)` in **both** near-identical files: `ckucns.c` (~2460)
  and `ckucon.c` (~2463). Note: `make linux` compiles `ckucns.c` (via the `xermit` target), not
  `ckucon.c`; the `ckucon.c` copy was verified by a separate test-compile with the `linux` flags
  (no new warnings) so both stay consistent.
- **V-10 — `spdtab` startup heap overflow from `sizeof`-a-pointer (ASan-confirmed).** `ckuus5.c`.
  Both required edits applied together: (a) the per-entry allocation at `:997` widened to a uniform
  `malloc(20 + 2)`; (b) in the sort block (~1030–1052) `int n = sizeof spdtab + 2;` (which yielded
  `n=10`) corrected to `int n = maxspeedlen + 2;`, with `speeds[i]`/`tmp[i].kwd` both `malloc(n)`
  and `ckstrncpy(speeds[i], spdtab[i].kwd, n)`.
- **V-13 — `SET KEY` `\K`-doubling overflow of global `line[]` (confirmed).** `ckuus3.c`
  `dosetkey()` (~6789). Bounded the doubling loop against `LINBUFSIZ - 1`, guarded each `line[y++]`
  store, and added the post-loop `if (y >= LINBUFSIZ - 1) { printf("?Key definition too long\n"); ... return (-9); }`
  error path; the `\{K`/`\K` two-character lookback is preserved at the truncation boundary.
- **V-14 — `OUTPUT`/`LINEOUT` quoting overflow of `line[]` (confirmed).** `ckuusr.c` (~8397). Same
  pattern: loop bounded, each write guarded, post-loop `if (y >= LINBUFSIZ - 1) { printf("?Output text too long\n"); return (success = 0); }`.
- **V-15 — unbounded completion-insertion copies into `cmdbuf`.** `ckucmd.c`. Added a bounded
  `bufcpy()` helper next to `addbuf()` (~7014) sharing its `cmdbuf + CMDBL` bound (without the
  trailing-space side effect), and replaced the three raw `while ((*bp++ = *sp++)); bp--;`
  completion loops (~2210, 2393, 2427) with `bufcpy(sp);`.
  **Deviation:** added a forward declaration `static void bufcpy(char *);` (`ckucmd.c:297`) because
  two of the three call sites precede the definition (required to compile under C23). Audited the
  file for other instances; the copy in `setatm()` is already pre-guarded and left unchanged.

## Phase 3 — High, conditional (`ee4e9b9`)

- **V-12 — recursive FTP GET/MGET path traversal via server-supplied names.** `ckcftp.c`. Added a
  `has_dotdot()` helper (rejects a `..` component or a leading separator, but not ordinary
  separators) with a forward declaration at `ckcftp.c:1352`. Guarded both server-name sites: the
  MLSD recursion (`ckcftp.c:12981`, before the CWD/recurse, skipping via the existing `again:`
  label) and the NLST local-file assembly (`ckcftp.c:6816`).
  **Deviation:** at the NLST site the refusal uses the function's own `skipthis++` / `msg = ...`
  idiom (matching the three sibling checks in that block), rather than a raw `goto`/`continue`.
- **V-11 — remote `!filename` local execution via `SET TRANSFER PIPES ON`.** Split the single
  `usepipes` flag into a send flag and a new receive flag so enabling outbound piping no longer
  silently authorizes inbound command execution:
  - `int usepipes_recv = 0;` (`ckcmai.c:780`, default off) with matching `extern` in `ckcfns.c:131`
    and `ckuus3.c:238`.
  - The receive path in `ckcfns.c` `rcvfil()` (~3462) now requires `usepipes && usepipes_recv` and
    prints a warning when a peer requests pipe execution. The two SEND-direction sites
    (`sinit()`, the `-s !cmd` CLI arg) remain gated on `usepipes` alone, as intended.
  - New keyword wired: `SET TRANSFER PIPES-RECEIVE ON|OFF` — `#define XYX_PIPRCV 14`
    (`ckuusr.h:982`), a `tstab[]` entry (`ckuus3.c` ~1366), a `case XYX_PIPRCV:` in `doprm()`
    (`ckuus3.c:11555`) mirroring `case XYX_PIP:` (including the `nopush` guard, calling
    `seton(&usepipes_recv)`), and a `Transfer Pipes-Receive:` line in `shoxfer()` /
    `SHOW TRANSFER` (`ckuus4.c`).
  - **Deviation:** the proposal's illustrative syntax was a *nested* `SET TRANSFER PIPES RECEIVE ON`.
    The applied form is a **flat, hyphenated sibling keyword** `PIPES-RECEIVE`, because every other
    `SET TRANSFER` option is a flat keyword; a nested sub-grammar would have been the only one of
    its kind in that table. The security-critical property (receive default **off**) is unchanged.
    The `SHOW TRANSFER` line was added; the denser hand-aligned columnar `SHOW PROTOCOL` dump in
    `shoparp()` was intentionally left untouched to avoid corrupting its fixed-column layout.

## Phase 4 — Medium (`6780ef3`)

- **V-16 — FTP client trusts PASV reply IP (SSRF-style redirect).** `ckcftp.c:11374`, `initconn()`.
  `data_addr.sin_addr = hisctladdr.sin_addr;` — ignore the address in the reply, always reconnect
  to the control-connection host; only the port is taken from the reply.
- **V-17 — `.netrc` parser overflow.** `ckcftp.c:13432`. Added
  `#define TOKVALMAX (tokval + sizeof(tokval) - 1)` and guarded every `*cp++ = c` in both copy
  loops with `if (cp < TOKVALMAX)`.
- **V-19 — predictable serial lock file without `O_EXCL` under `priv_on()`.** `ckutio.c:3907`,
  `ttlock()`. `creat(tmpnam, 0444)` → `open(tmpnam, O_WRONLY | O_CREAT | O_EXCL, 0444)`.
- **V-20 — `SET MODEM DIAL-COMMAND` used as `printf` format.** `ckudia.c`. Added a `dialexpand()`
  helper (`:4703`) that substitutes the phone number for the first literal `%s` and copies any
  other `%` through as text; replaced `sprintf(lbuf, dcmd, xnum)` with
  `dialexpand(lbuf, dcmd, xnum, sizeof(lbuf))` (`:5450`).
- **V-21 — fd leak on overwrite.** `ckufio.c:1217`, `zopeno()`. Added `else { close(fd); }` on the
  non-tty probe path so the `isatty()` probe descriptor is not leaked.
- **V-22 — no `O_NOFOLLOW`/atomicity between `zchko()` and `zopeno()`.** `ckufio.c`. The non-tty
  write open was restructured from `fopen()` to
  `open(name, O_WRONLY|O_CREAT|(append?O_APPEND:O_TRUNC)|O_NOFOLLOW, 0600)` + `fdopen()`
  (`:1229`), with close-on-`fdopen`-failure; `O_NOFOLLOW` also added to the `zchko()` probe
  (`:2275`). The tty-output path (`fdopen(filefd, p)`) is unchanged.
  See **Residuals** for the verification caveat.
- **V-23 — OOB read `table[-1]` in `nlookup()` under DEBUG.** `ckucmd.c:7859`. Debug line now logs
  `this` (the current, always-valid entry) instead of `table[lastmatch].kwd` (`lastmatch` is `-1`
  until a match).
- **V-24 — `filbuf` NUL-terminated at `CKMAXPATH` (`-DNOBIGBUF`).** `ckucmd.c` (~2258/2262).
  Bounded to `sizeof(filbuf)` / `sizeof(filbuf)-1` instead of `CKMAXPATH`.
- **V-25 — off-by-one heap write in `\s(name[n.])`.** `ckuus4.c:15412`, `zzstring()`. Guarded
  `q[x1+1] = NUL;` with `if (x1 + 1 <= k)` (q is `malloc(k+1)`); asking for one more than exists
  now returns the full string instead of writing one past.
- **V-26 — unguarded 128-entry table index in `xeglg()`.** `ckuxla.c:5396`. Added
  `if (c & 0x80) { return (UNK); }` before the `yeglg[c]` index, matching sibling translators.
- **V-27 — `SET EDITOR`/`SET BROWSER /OPTIONS` used as `printf` format.** `ckuusr.c`. Added a
  `pctsexpand()` `%s`-only helper (`:6338`) and replaced both `sprintf(tmpbuf, opts, arg)` calls —
  `doedit()` (`:6421`) and `dobrowse()` (`:6466`).
  **Note:** `dobrowse()` uses `browsopts`/`browsurl` (the proposal's `editopts`/`editfile` was
  shorthand); adapted accordingly.
- **V-28 — `setword()` partial-init NULL deref under OOM.** `ckclib.c` (~2883). On the second
  `malloc` failure, added `free(wordarray); wordarray = NULL;` before `return`, so the
  `if (!wordarray)` re-init guard stays honest on the next call.

## Known residuals / follow-ups (not addressed here)

1. **V-11 adjacent surface (server-side `GET !command`).** `sgetinit()` in generated
   `ckcpro.c` (from `ckcpro.w`) still lets a server run a remote-selected command and stream its
   output back, gated on `usepipes` alone — though it has its own independent `en_hos` permission
   gate on top. This is a *different* direction than V-11 (remote-selected command whose output is
   sent out, vs. a received file piped to a command's stdin) and outside the documented finding.
   Closing it would need a `ckcpro.w` edit + `make wart && make ckcpro.c`. Candidate follow-up.
2. **V-22 `O_NOFOLLOW` symlink rejection not proven end-to-end.** The create/overwrite/append
   mechanics are verified, and `O_NOFOLLOW` is correct by inspection, but the actual
   symlink-refusal behavior lives on the protocol *receive* path and was not exercised end-to-end
   (that needs a loopback two-instance transfer through a symlinked destination). Runtime proof is
   partial.
3. **V-27 helper placement edge case.** `pctsexpand()` sits under the `#ifndef NOPUSH`/`#ifndef NOFRILLS`
   nest before `doedit()`; `dobrowse()` (its other caller) is `#ifdef BROWSER`. For every buildable
   config in this fork these compile together (`BROWSER` implies `NOPUSH` is off; `NOFRILLS` is
   never defined here). A hypothetical manual `-DNOFRILLS` override — out of scope for this fork —
   would need the helper hoisted.
