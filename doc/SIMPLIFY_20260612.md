# Codebase Simplification Report — Removal of Windows, OS/2, and VMS Support

**Date:** 2026-06-12
**Tree:** C-Kermit 10.0 Beta.12 (`/Users/kenji/src/cpp-kermit`)
**Scope:** Remove all Windows (NT/Win32/Kermit-95), OS/2, and VMS code. Linux, macOS, and generic Unix support untouched. Plan 9 (`ckpker.mak`), Android (`android.mk`), and 2.11BSD (`ckubs2.mak`) files kept as instructed.

Build verification on macOS: **success** (see [Verification](#verification)). The resulting
`wermit` binary is byte-for-byte the same size as the pre-removal baseline build
(2,456,584 bytes) with an identical compiler-warning count (27), confirming that only
dead (never-compiled-on-macOS) code was removed.

A full backup of the original tree was made before any change:
`/tmp/cpp-kermit-backup-20260612.tar.gz`

## Method

1. **Analysis** was performed by a `c-expert` agent, which produced the file-deletion
   list, the exact `unifdef` symbol classification, and the risk list below.
2. **Whole-file deletions** of VMS-only and Windows-only files.
3. **Conditional-block removal** with `/usr/bin/unifdef -m` (in-place), in two passes,
   with per-file error checking (no parse errors occurred).
4. **Rebuild and smoke test** on macOS.

## Files deleted (16)

VMS-only sources, headers, and build files — none referenced by the Unix `makefile`,
and every `#include` of them from kept files sat inside a removed `#ifdef VMS` block:

- `ckvcon.c`, `ckvcvt.c`, `ckvfio.c`, `ckvioc.c`, `ckvioc.h`, `ckvold.c`,
  `ckvrms.h`, `ckvrtl.c`, `ckvrtl.h`, `ckvtio.c`, `ckvvms.h` (VMS C sources/headers, 15,025 lines incl. `ckcxxx.h`)
- `ckvker.com`, `ckvold.com` (DCL build scripts), `ckvker.mms` (MMS build file),
  `ckvker.hlp` (VMS help) — 4,034 lines
- `ckcxxx.h` — Kermit-95 GUI variable declarations; zero includers anywhere in the tree

## Preprocessor symbols removed (unifdef)

**Pass 1 — treated as never defined (`-U`):**

- VMS: `VMS`, `__VMS`, `VMSV80`, `VMSV70`, `OLD_VMS`, `VMS64BIT`
- VMS TCP stacks: `DEC_TCPIP`, `CMU_TCPIP`, `WINTCP`, `UCX50`
- OS/2: `OS2`, `OS2ONLY`, `OS2K95`
- Windows: `NT`, `NTSIG`, `NTASM`, `CKT_NT`, `WIN32`, `_WIN32`, `_WIN64`,
  `WINVER`, `KUI`, `K95`, `K95BUILD`
- Compound macros false on Unix: `OS2ORVMS`, `OS2ORWIN32`, `OS2ORWINDOWS`

**Pass 1 — treated as always defined (`-D`), because `ckcdeb.h` defines them on every
Unix build** (getting these backwards would have deleted live Unix code):

- `OS2ORUNIX`, `VMSORUNIX`, `WIN32ORUNIX`, `UNIXOROSK`, `OSKORUNIX`

**Pass 2 — residual feature macros (`-U`)**, found by post-pass scan and individually
verified never defined in any remaining file or in the `makefile`:

- `OS2MOUSE`, `OS2PM` (OS/2 mouse / Presentation Manager)
- `NTLM`, `NT_TCP_OVERLAPPED`, `NTSIGX` (Windows NTLM auth / overlapped I/O / signals)
- `VMSCURSE` (VMS curses)
- `MULTINET`, `TGVORWIN`, `CK_TGV_AXP` (VMS MultiNet TCP stack and derivatives)

**Deliberately untouched:**

- `CK_SMG` (VMS Screen Management): its only `#define` was inside a removed
  `#ifdef VMS` block, after which the whole `#ifdef CK_SMG` region in `ckuusx.c`
  (which `#include`d the deleted `ckvvms.h`) became provably dead and was stripped
  with a targeted `unifdef -UCK_SMG ckuusx.c`.
- `NTELOPTS` ("number of telnet options") and the `NTLM_*` telnet auth-type constants
  in `ckctel.h` — look Windows-ish but are not.
- Derived feature macros (`BIGBUFOK`, `CK_CURSES`, `MYCURSES`, `DCMDBUF`, …) — resolve
  correctly per platform at compile time.
- Other obsolete non-Unix platforms outside the requested scope (`MAC` classic
  Macintosh, `AMIGA`, `OSK`, `datageneral`, `BEBOX`, `STRATUS`, …) remain as
  dead `#ifdef` blocks.

## Makefile

No edits were required: the Unix `makefile` never compiled any `ckv*` file, its
