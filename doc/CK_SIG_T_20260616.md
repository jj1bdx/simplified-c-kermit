# Add `ck_sig_t` typedef and convert `void (*x)(int)` signal-handler declarations

Date: 2026-06-16

## Motivation

After the 2026-06-15 `signal()`→`ck_signal()`/`sigaction()` refactor, signal-handler
pointers are spelled out verbatim as `void (*x)(int)` throughout the tree — in save/restore
variables, function parameters, and the `ck_signal`/`conint` prototypes. This is verbose and,
for the function-returning-pointer-to-function forms (`void (*ck_signal(...))(int)`), hard to
read. A single named type makes every site legible.

## Change

A new typedef in the master header `ckcdeb.h` (included first in every `*.c`/`*.h` via the
pinned `/* clang-format off */` first-include island), placed just above the `ck_signal`
comment/prototype block so it precedes all uses:

```c
/* Signal-handler pointer type: void function of one int, returning void. */
typedef void (*ck_sig_t)(int);
```

Then every verbatim `void (*NAME)(int)` spelling of a signal-handler pointer was rewritten to
`ck_sig_t NAME` (abstract-declarator prototype params → bare `ck_sig_t`). Because `ck_sig_t`
**is** `void (*)(int)` — function-pointer type compatibility in C is structural, independent of
the typedef name — this is a pure source-spelling change with no effect on the type system or
codegen on any `#ifdef` path.

**Note:** the request named `ckudeb.h`, which does not exist; the master header is `ckcdeb.h`.

### Conversion inventory (exhaustive)

| File | Sites | Kind |
|------|-------|------|
| `ckcdeb.h` | 1 + 2 | new typedef; `ck_signal` and `conint` prototypes |
| `ckusig.c` | 3 | `ck_signal` definition (return type + param); two `savhandler` vars |
| `ckutio.c` | 12 | save/restore statics (`saval`,`savquit`,`savusr1/2`,`savpipe`,`savdanger`,`jchdlr`,`save_sigchld`), `occt`, `osigint`, `conint` params, `istat`/`qstat` multi-declarator |
| `ckufio.c` | 1 | `istat`/`qstat` multi-declarator |
| `ckudia.c` | 2 | `savalrm`, `savint` statics |
| `ckuus4.c` | 2 | `oldsig` vars |
| `ckuus6.c` | 1 | `oldsig` var |
| `ckuscr.c` | 1 | `savealm` var |
| `ckcsig.h` | 1 | `typedef void (*ck_sighand)(int);` → `typedef ck_sig_t ck_sighand;` |

The two multi-declarator collapses are exact (both declarators were `void (*)(int)`):
`void (*istat)(int), (*qstat)(int);` → `ck_sig_t istat, qstat;`.

The `ck_signal` return type changes from the function-returning-function-pointer spelling
`void (*ck_signal(int type, void (*)(int)))(int)` to `ck_sig_t ck_signal(int type, ck_sig_t)`;
these denote the identical function type.

### Deliberately left unconverted

- `ckcsig.h:15` `typedef void (*ck_sigfunc)(void *);` — signature is `(void *)`, not `(int)`.
- `ckcftp.c:587` `typedef void (*sig_t)(int);` — deliberately mirrors the platform `sig_t`
  type as a fallback (per the adjacent essay); a type-name compatibility definition, not a
  variable. `ckcftp.c` is entirely untouched. `sig_t` and `ck_sig_t` remain distinct names for
  the identical type; cross-assignment from `ck_signal()` (returns `ck_sig_t`) into `ckcftp.c`'s
  `sig_t oldintr`/`oldintp` and the `struct _ftpsnd`/`_ftprecv` members stays valid.
- `ckcftp.c:558` — inside the historical `/* … */` essay comment.
- `ckcdeb.h:768` `ckntsignal` — inside `#ifdef CKNTSIG` (never defined; documented inactive
  block); harmless dead code.

## Verification

`ck_sig_t` is by construction the same type as `void (*)(int)`, so the change is provably
codegen-neutral. Verified at every level:

- **L0 — Byte-identical build (strongest gate).** Fixed `SOURCE_DATE_EPOCH=1700000000`, fully
  clean on both sides (`make clean && make wart && make ckcpro.c && make linux`): the whole
  `wermit` binary is **`cmp`-identical** before vs after, and **all 29 object files are
  byte-identical**. Build is warning/error-clean; `make check` = SUCCESS.
- **L1 — Signal runtime tests.** `./wermit -B -C "pause 2, exit"` → Elapsed **0:02.00** (the
  SIGALRM `longjmp` timer still fires); SIGINT delivered mid-`PAUSE 10` aborts (`^C...`) and
  returns to the command loop (the SIGINT save/restore handlers — now `ck_sig_t` — still
  install/fire/restore); clean startup/exit.
- **L2 — macOS/BSD `#ifdef` paths.** Preprocessing `ckcdeb.h` with `-DBSD44 -DMACOSX -DMACOSX10`
  yields the converted prototypes with `ck_sig_t` resolved (the typedef is unconditional);
  `cc -fsyntax-only` on the BSD/macOS path of the converted files shows no `ck_sig_t`,
  brace, declarator, or redefinition errors.
- **L3 — c-expert review.** Verdict: **semantically equivalent, exactly behavior-preserving.**
  Confirmed (a) `ck_sig_t` is the identical type at every replaced site incl. the `ck_signal`
  return type and abstract-declarator params (compile-tested the abstract-param, multi-declarator
  collapse, and cross-typedef `ck_sig_t→sig_t` assignment cases with zero diagnostics); (b) no
  conversion captured a differently-typed pointer (`ckctel.c`'s `int (*)(int)` parser callbacks
  and `ck_sigfunc` correctly untouched); (c) no saved-pointer comparison or invocation relied on
  the spelled-out form (`== SIG_IGN`/`SIG_DFL`, `(*oldintr)(SIGINT)`, `oldintr != cmdcancel`);
  (d) no `ckcsig.h` ordering hazard (the first-include pin guarantees `ck_sig_t` precedes
  `ck_sighand`); (e) the out-of-scope decisions are correct; (f) a whole-tree grep found nothing
  missed.
