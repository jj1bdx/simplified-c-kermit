# Codebase Simplification Report — Removal of 26 Dead Platform Flags

**Date:** 2026-06-13
**Tree:** C-Kermit 10.0 Beta.12 (`/home/kenji/src/simplified-c-kermit`)
**Scope:** Treat 26 obsolete platform/compiler selection macros as *permanently
undefined* and delete the dead conditional code they gate. These flags only ever
arrived via `-D` on now-obsolete makefile targets for hardware/OS platforms that this
tree no longer supports. Linux, macOS, and the remaining BSD/Unix platform code are
**unaffected**.

## Flags removed (26)

```
AS400  __386BSD__  C70  CIE  SINIX  SNI544  SNI543  SNI541  POWERMAX  ICL_SVR4
XF68R3V6  XF88R32  IX370  PCIX  sxaE50  RTU  TRS16  _386BSD  OU8  UW7  NEUTRINO
LYNXOS  UTS24  VXVE  ZILOG  AMIX
```

These name dead targets such as AS/400, 386BSD, Convergent CIE, Siemens-Nixdorf
SINIX/SNI, Concurrent PowerMAX, ICL SVR4, X/Open XF68/XF88, IBM IX370/PCIX, Stratus
sxaE50, Masscomp/Concurrent RTU, TRS-16/Xenix, Olivetti OU8, UnixWare 7, QNX Neutrino,
LynxOS, Amdahl UTS, Stratus VOS VXVE, Zilog ZEUS, and Amiga UNIX (AMIX).

## Flags deliberately KEPT — `SVR4`, `SVR3`, `ATTSV`

The original request also listed `SVR4`, `SVR3`, and `ATTSV`. These were **excluded**
because they are **not** Linux-only dead gates — they are live on macOS and all BSD
builds through an implication chain in `ckcdeb.h`:

```
MACOSX10  →  MACOSX  →  BSD44  →  SVR4  →  SVR3 / ATTSV
```

(`ckcdeb.h:151-161, 590-621`; the `bsd44`/`freebsd*`/`netbsd*`/`openbsd*` targets reach
the same chain via `-DBSD44`.) On macOS/BSD these three gate live serial-I/O,
file-I/O, and networking code, and `ckuusx.c:2501-2504 / 2588-2591` performs a
deliberate BSD44 save/restore of `ATTSV`. Treating them as undefined would silently
break the still-supported macOS/BSD builds, so they were left fully intact. This was
confirmed with the maintainer before proceeding.

## Method

1. **Verification of the premise.** Preprocessing `ckcdeb.h` with the exact `make
   linux` flags (`-std=gnu17 -DLINUX -DFNFLOAT -DCK_POSIX_SIG -DCK_NEWTERM
   -DTCPSOCKET -DLINUXFSSTND -DNOCOTFMC -DPOSIX -DUSE_STRERROR`) confirmed all 26
   flags resolve **undefined**, while `POSIX`, `SVORPOSIX`, `SVR4ORPOSIX`, and `UNIX`
   resolve **defined** and must be preserved. None of the 26 is ever `#define`d
   anywhere in `*.c *.h *.w`; each appears only in plain `#ifdef`/`#ifndef` (no
   compound `defined(...)` expressions), so `unifdef` performs pure mechanical block
   selection with no `-D` traps.

2. **Conditional-block removal** with `/usr/bin/unifdef -m` over the 13 affected files,
   passing `-U` for each of the 26 flags, with per-file return-code checking
   (`unifdef` exits 0 = unchanged, 1 = changed, 2 = error). Result: **13 changed, 0
   errors**. The generated `ckcpro.c` was excluded per project convention; in fact
   none of the 26 flags appears in `ckcpro.w`/`ckcpro.c`, so no `wart` regeneration was
   required.

3. **Build + smoke test** on Linux (`rm -f wermit && make linux`, then `make check`).

### Implication-chain safety

The one structural risk was the `BSD44 → SVR4 → SVR3/ATTSV` chain and the `ckuver.h`
HERALD ladders. Because `SVR4`/`SVR3`/`ATTSV` were kept, those blocks remain
syntactically valid and functional for macOS/BSD. Post-edit checks confirmed:

- `gcc -E -DMACOSX10` and `gcc -E -DBSD44`: `SVR4`, `SVR3`, `ATTSV` all still
  **defined**.
- No orphaned or contradictory `#define`/`#undef` of any of the 26 left behind.
- Zero remaining `#if*` conditionals referencing any of the 26.

## Notes

- **No hand-fixes were needed.** `unifdef -m` produced a tree that built cleanly on the
  first attempt; no dangling `#else`/`#endif` or orphaned declarations.
- **`SHOW FEATURES` output** loses the dead-platform tokens (e.g. `CIE`, `RTU`,
  `ZILOG`) that were wrapped in the removed `#ifdef` blocks of `ckuus5.c` — cosmetic
  and expected.
- **Inert textual references** (comments, e.g. `ckutio.c` "RTU treats ^Z as EOF") that
  fall outside removed conditional arms were left untouched; none is a live code path.
