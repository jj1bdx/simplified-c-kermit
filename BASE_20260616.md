# BASE_20260616 — what changed from `initial-without-windows-or-vms` to `base-20260616`

Date: 2026-06-16

## Scope

This document summarizes, at a glance, how the **base** tree differs from the **initial** tree:

| Tag | Commit | State |
|-----|--------|-------|
| `initial-without-windows-or-vms` | `5a8fc5a` | the very first commit — the tree right after Windows/OS2/VMS support was stripped (see `SIMPLIFY_20260612.md`, already present at this tag). |
| `base-20260616` | `2eab866` | current HEAD. |

**60 commits** separate them — the 2026-06-12 → 2026-06-16 simplification campaign. The premise
throughout: *this is now a Linux / macOS / Unix-only tree*, so anything that only existed to support
other platforms, pre-ANSI compilers, or removed features is dead weight.

`git diff --stat` headline: **100 files changed, 186,666 insertions(+), 232,919 deletions(-)**
(20 deleted, 28 added, 52 modified). The insertion/deletion totals are inflated by whole-file
`clang-format` reformatting on both sides — they do **not** reflect 186k lines of new logic; almost
all of it is whitespace/brace churn and mechanical text substitution (see §6). Per-change detail
lives in the report files cross-referenced in §8; this is the consolidated overview.

## 1. Platform & feature removals

- **Android** removed (`SIMPLIFY_20260612_2.md`): `android.mk`, `android.txt`, and the `#ifdef`
  blocks.
- **SSL/TLS + Kerberos** removed (`SIMPLIFY_20260613.md`): the entire Telnet
  authentication/encryption stack (KRB4, SRP, DES/CAST). Deleted the `ck_*` security sources and
  `ckuath.*`/`ckuat2.h`; dropped all `+ssl`/`+krb*`/`+srp` makefile targets.
- **Non-ANSI / K&R C** removed (`SIMPLIFY_20260613_2.md`): `CK_ANSIC` is now assumed always defined;
  all pre-ANSI fallback branches stripped.
- **26 obsolete platform `-D` macros** removed from the C source (`SIMPLIFY_20260613_3.md`):
  dead `#ifdef` branches for `AS400`, `SINIX`, `LYNXOS`, `NEUTRINO`, `AMIX`, … deleted with
  `unifdef`.
- **Dead makefile targets** removed in waves (`SIMPLIFY_20260613_4.md` … `_6.md`): the 38 targets
  defining those macros, then 64 Xenix/ancient-BSD targets, then *all* remaining foreign-Unix
  families (IRIX, HP-UX, AIX, SCO/Unixware, Solaris/SunOS, Ultrix/OSF/Tru64, Sequent, NeXT, QNX,
  DG/UX, …) and the orphaned generic bases. **`makefile`: 9,520 → 1,187 lines.** Only Linux, macOS,
  and modern-BSD (FreeBSD/NetBSD/OpenBSD/MirBSD) **build targets** survive.
- **`#ifdef COMMENT` dead-code blocks** removed (`SIMPLIFY_SUMMARY_20260613.md`): `COMMENT` was never
  defined, so these 731 guards were C-Kermit's "comment out a region" idiom — all unreachable.
  ~6,300 lines removed; the build was provably unchanged.

## 2. Source modernization (behavior-preserving)

- **`_PROTOTYP()` expanded and removed** (`SIMPLIFY_20260615.md`): all 1,175 pre-ANSI
  prototype-forwarding macros expanded to plain ANSI prototypes; the macro deleted.
- **`VOID` / `CKVOID` / `CONST` aliases expanded and removed** (`PROTOTYP_PLAN_20260615_2.md`):
  688 substitutions to the plain `void`/`const` keywords; no portability alias macros remain.
- **`SIGTYP` / `SIGRETURN` expanded and removed** (`SIGTYPE_PLAN_20260615.md`): the signal
  return-type macros resolved to `void`/`return` (correct on every supported platform — POSIX on
  Linux, SVR3 via the MACOSX→BSD44→SVR4→SVR3 chain on macOS/BSD).
- **`signal()` save/restore → `sigaction()` via `ck_signal()`** (`SIGACTION_PLAN_20260615.md`,
  `SIGACTION_MACOS_20260615.md`): a drop-in wrapper using `SA_RESTART` + `sigemptyset`, faithfully
  reproducing glibc/BSD/macOS `signal()` semantics; verified on real macOS hardware.
- **`ck_sig_t` typedef + `void (*)(int)` conversion** (`CK_SIG_T_20260616.md`): added
  `typedef void (*ck_sig_t)(int);` to `ckcdeb.h` and rewrote the verbose signal-handler-pointer
  spellings; pure source-spelling, byte-identical build.
- **`ckuver.h` herald detection flattened** (`SIMPLIFY_SUMMARY_20260613.md` §3): the nested
  `#else` ladder became a flat first-match-wins `#ifndef HERALD` guard sequence (macOS > OpenBSD >
  NetBSD > FreeBSD > 4.4BSD; Linux separate) — cosmetic, clang-format-friendly.
- **C23 signal-type fix + `-std=gnu17` dropped** (`SIGNAL_TYPE_20260615.md`): empty-paren signal
  pointers gained explicit `(int)` prototypes for the compiler's default (C23) standard.

## 4. Editor / format tooling

- **`clang-format`**: `InsertBraces` enabled (single-statement bodies braced — the one non-cosmetic
  transform; see `INSERT_BRACES_20260615.md`) and `SortIncludes` enabled
  (`ENABLE_SORTINCLUDES_20260615.md`), with **`ckcdeb.h` pinned as the first include everywhere**
  via `/* clang-format off */` islands (`MOVE_CKCDEB_H_20260615.md`). All `*.c` reformatted;
  headers reformatted (`FORMAT_C_PLAN_20260614.md`, `FORMAT_H_PLAN_20260614.md`).
- **clangd / clang-tidy** editor config added (`.clangd`, `.clang-tidy`) mirroring the `linux`
  target's flags; editor-only, cannot affect the build.

## 5. Added files

The 28 added files are almost entirely **documentation** — the `SIMPLIFY_*.md`, `OS_*.md`,
`FORMAT_*.md`, `SIG*.md`, `UBUNTU_TEST_*.md`, and `CK_SIG_T_20260616.md` report files that record
each change — plus `.clang-tidy`. No new C source was introduced.

## 6. Magnitude

- `makefile`: **9,520 → 1,187 lines** (−87%).
- 20 source/build files deleted; 0 new C sources; 28 docs/config added; 52 files modified.
- The ~187k/233k insertion/deletion totals are dominated by `clang-format` reformatting (whitespace
  + InsertBraces) layered on the mechanical `_PROTOTYP`/`VOID`/`CONST`/`SIGTYP`/`ck_sig_t`
  substitutions and dead-`#ifdef` removal. A c-expert token-level audit of the highest-churn files
  (`ckcuni.c`, `ckutio.c`, `ckcftp.c`) found **no live-path logic edits** — every removed code chunk
  sat inside a dead `#ifdef COMMENT`, an obsolete-platform branch, or a removed-feature
  (`FTP_GSSAPI`/KRB) guard.
- The retired foreign-Unix makefile **stanzas** are gone, but some stale foreign-Unix **comment**
  prose (AIX/HP-UX/SCO/NeXT/SunOS acknowledgement and section headers) remains as harmless
  recipe-less text.

## 7. Base verification status

The `base-20260616` tree builds clean: `make linux` is **warning-clean**, `make check` =
**SUCCESS**, binary `wermit` = **2,859,968 bytes**. Each behavior-preserving modernization was
verified in its own report at the strongest applicable level — whole-binary byte-identical rebuilds
(`SOURCE_DATE_EPOCH`-fixed, fully clean) for the pure source-spelling substitutions
(`_PROTOTYP`/`VOID`/`CONST`/`ck_sig_t`), per-object `.o` + token proofs for the brace-touching
reformat, and object-level + runtime + on-hardware-macOS verification for the `sigaction()` wrapper.
The `#ifdef COMMENT` and foreign-platform removals never compiled on the surviving targets, so the
build is unchanged by construction.
